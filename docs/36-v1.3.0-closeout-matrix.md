# 36 v1.3.0 Closeout Matrix

_Last reviewed: 2026-03-25 (baseline commit f7e6cff)._

This matrix defines the concrete closeout sequence from `v1.2.5` into `v1.3.0`.

## Entry Conditions

All items must be true before running `v1.3.0` release preflight:

1. `v1.2.5` release-candidate strict gate is green:
: `docs/assets/demo/v1.2.5/m10-status.json` has `strictReady: true`.
2. Canonical release-candidate pass artifacts are materialized:
: `m10-v130-release-candidate-pass.json`,
: `m10-v130-release-candidate-capture.json`,
: `m10-digest-summary.json`,
: `m10-event-slices.json`.
3. Working tree is clean.

## Runtime Evidence Step

1. In browser console, run:
```js
window.obscurAppEvents?.clear?.();
copy(window.obscurM10TrustControls?.runV130ReleaseCandidateCaptureStabilizedJson?.({
  eventWindowSize: 400,
  expectedStable: true,
  settlePasses: 2,
}));
```
2. Save JSON to:
: `docs/assets/demo/v1.2.5/raw/m10-v130-release-candidate-capture.json`.
3. Materialize/check/report:
```bash
pnpm demo:m10:rc:materialize -- --capture docs/assets/demo/v1.2.5/raw/m10-v130-release-candidate-capture.json
pnpm demo:m10:rc:check
pnpm demo:m10:rc:status
pnpm demo:m10:rc:next
```

## Closeout Validation Pack

Run these on clean `main`:

1. `pnpm version:check`
2. `pnpm docs:check`
3. `pnpm --dir apps/pwa exec vitest run app/shared/m10-trust-controls-bridge.test.ts`
4. `pnpm --dir apps/pwa exec tsc --noEmit --pretty false`
5. `pnpm release:test-pack -- --skip-preflight`
6. `pnpm release:preflight -- --tag v1.3.0`

Automation shortcut:
1. `pnpm closeout:v130:check`
: runs strict RC artifact checks + docs/version + focused M10 test/typecheck
: without rewriting `m10-status.json` timestamps,
: and enforces a clean working tree unless `--allow-dirty` is passed.
2. `pnpm closeout:v130:check:refresh-status`
: same as above, but also refreshes `m10-status.json` via `demo:m10:rc:status`.
3. `pnpm closeout:v130:check:with-preflight`
: includes `release:preflight -- --tag v1.3.0`.

## Acceptance Snapshot

Use this compact acceptance checklist before tag push:

1. `m10-status.json`:
: `strictReady === true`.
2. `m10-v130-release-candidate-pass.json`:
: `releaseCandidateGate.pass === true`.
3. `m10-event-slices.json`:
: all required buckets (`cp2`, `cp3Readiness`, `cp3Suite`, `cp4Closeout`, `v130Closeout`, `v130Evidence`, `v130ReleaseCandidate`) are non-empty.
4. `m10-digest-summary.json`:
: `summary.m10TrustControls` is non-null.
5. Release preflight command exits with code `0`.

## Release Step

After acceptance is complete:

1. `git tag v1.3.0`
2. `git push origin v1.3.0`
3. publish GitHub Release notes from `CHANGELOG.md`.
