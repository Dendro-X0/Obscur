# v0.9 Discovery Overhaul Spec

_Last reviewed: 2026-03-07 (baseline commit ab08104)._

## Scope

This wave introduces a multi-source discovery stack to replace relay-only global search behavior.

Core decisions:

1. Contact-card-first entry for adding people.
2. Source model: local cache + relay + optional index service.
3. Discoverability remains private by default.
4. Legacy invite/search paths stay compatible during rollout.
5. Community discovery uses the same engine with a community intent/ranking path.

## Engine Architecture

`apps/pwa/app/features/search/services/discovery-engine.ts` provides:

1. `QueryPlanner.classifyQuery(...)`
2. `ResultMerger.mergeResults(...)`
3. `DiscoveryEngine.run(...)` with phase/state callbacks

Phases:

1. `idle`
2. `running`
3. `partial`
4. `complete`
5. `timeout`
6. `offline`
7. `degraded`

Reason codes:

1. `invalid_input`
2. `no_match`
3. `offline`
4. `relay_degraded`
5. `index_unavailable`
6. `canceled`

## Source Adapters

1. Local source:
   - profile/invite cache lookup
   - local community seeds from joined groups
2. Relay source:
   - people/invite via `relay-discovery-query`
   - communities via `relay-community-discovery-query`
3. Index source (optional):
   - `GET /v1/discovery/people?q=...`
   - `GET /v1/discovery/communities?q=...`
   - only used if `NEXT_PUBLIC_DISCOVERY_INDEX_URL` is configured

## Contact Card Model

`apps/pwa/app/features/search/services/contact-card.ts`:

1. Encodes/decodes `ContactCardV1`
2. Supports deep-link extraction (`obscur://contact?card=...`)
3. Supports optional signature using crypto invite signing APIs
4. Includes verifier path used during contact-card resolution

## Deterministic Add Path

1. Added Friend Code v2 encode/decode with checksum and pubkey payload.
2. Added identity resolver pipeline with strict order:
   - Contact Card
   - Friend Code v2
   - npub/hex
   - legacy `OBSCUR-*` compatibility bridge
   - text marked unsupported for deterministic add
3. Added resolved identity cache with one-time migration seeding from existing discovery cache.

## Request Outbox

1. Added persistent contact request outbox hook with status state machine:
   - `queued`, `publishing`, `sent_partial`, `sent_quorum`, `accepted`, `rejected`, `failed`
2. Outbox uses existing connection-request send pipeline and applies retry with backoff.
3. Add Friend UI now separates:
   - Resolve identity
   - Queue/send request
   - Observe outbox status

## UX Surfaces

`/search` now exposes three explicit surfaces:

1. Add Friend (default)
2. People
3. Communities

UI behavior:

1. Immediate partial rendering when local/index returns quickly.
2. Progressive completion while relay queries continue.
3. Explicit degraded/offline banners with reason codes.
4. Source status chips for local/relay/index.
5. Share panel for contact card (QR + link + short code copy).

## Compatibility

1. Legacy `OBSCUR-*` invite code resolution remains supported.
2. Existing profile metadata invite embedding remains valid.
3. Existing search hook API remains callable; it now returns discovery query state.
