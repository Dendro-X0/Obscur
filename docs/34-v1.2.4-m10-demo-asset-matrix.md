# 34 v1.2.4 M10 Demo Asset Matrix

_Last reviewed: 2026-03-25 (baseline commit d84812de)._

This matrix defines the demo-asset bundle required while `v1.2.4` is the active M10 closeout lane.

Goal:
1. keep roadmap execution evidence-backed,
2. produce repeatable demo assets from canonical owners only,
3. avoid manual drift between runtime truth and release storytelling.

## Output Location

Store all generated files in:
- `docs/assets/demo/v1.2.4/`

Automation:
1. initialize missing template files:
: `pnpm demo:m10:init`
2. structure-only verification (safe before runtime captures):
: `pnpm demo:m10:check:structure`
3. materialize canonical pass-lane files from raw captures (preferred one-shot bundle):
: `pnpm demo:m10:materialize -- --bundle docs/assets/demo/v1.2.4/raw/m10-v124-demo-bundle.json`
4. split capture fallback materialization:
: `pnpm demo:m10:materialize -- --v130-evidence docs/assets/demo/v1.2.4/raw/m10-v130-evidence-capture.json --digest-bundle docs/assets/demo/v1.2.4/raw/m10-digest-event-bundle.json`
5. strict verification (requires real pass-lane captures):
: `pnpm demo:m10:check`
6. status report export (writes readiness summary):
: `pnpm demo:m10:status`
: output: `docs/assets/demo/v1.2.4/m10-status.json`

## Console Capture Commands

Run on a clean replay window when possible:

```js
window.obscurAppEvents?.clear?.();
```

Pass-lane capture commands:

```js
copy(window.obscurM10TrustControls?.runV124DemoAssetBundleCaptureJson?.({ eventWindowSize: 400, expectedStable: true }));
copy(window.obscurM10TrustControls?.runCp3ReadinessCaptureJson?.({ eventWindowSize: 400, expectedStable: true }));
copy(window.obscurM10TrustControls?.runCp3SuiteCaptureJson?.({ eventWindowSize: 400, expectedStable: true }));
copy(window.obscurM10TrustControls?.runCp4CloseoutCaptureJson?.({ eventWindowSize: 400, expectedStable: true }));
copy(window.obscurM10TrustControls?.runV130CloseoutCaptureJson?.({ eventWindowSize: 400, expectedStable: true }));
copy(window.obscurM10TrustControls?.runV130EvidenceCaptureJson?.({ eventWindowSize: 400, expectedStable: true }));
```

Save the copied JSON string to:
1. `docs/assets/demo/v1.2.4/raw/m10-v124-demo-bundle.json` (preferred one-shot bundle),
2. `docs/assets/demo/v1.2.4/raw/m10-v130-evidence-capture.json` (split fallback).

Compact digest + event bundle:

```js
copy(JSON.stringify((() => {
  const digest = window.obscurAppEvents?.getCrossDeviceSyncDigest?.(400);
  return {
    summary: {
      incomingRequestAntiAbuse: digest?.summary?.incomingRequestAntiAbuse ?? null,
      uiResponsiveness: digest?.summary?.uiResponsiveness ?? null,
      m10TrustControls: digest?.summary?.m10TrustControls ?? null,
    },
    events: {
      cp2: digest?.events?.["messaging.m10.cp2_stability_gate"] ?? [],
      cp3Readiness: digest?.events?.["messaging.m10.cp3_readiness_gate"] ?? [],
      cp3Suite: digest?.events?.["messaging.m10.cp3_suite_gate"] ?? [],
      cp4Closeout: digest?.events?.["messaging.m10.cp4_closeout_gate"] ?? [],
      v130Closeout: digest?.events?.["messaging.m10.v130_closeout_gate"] ?? [],
      v130Evidence: digest?.events?.["messaging.m10.v130_evidence_gate"] ?? [],
    },
    recentWarnOrError: digest?.recentWarnOrError ?? [],
  };
})(), null, 2));
```

Save the copied JSON string to:
1. `docs/assets/demo/v1.2.4/raw/m10-digest-event-bundle.json`.

Materialize canonical demo assets from one-shot bundle:

```bash
pnpm demo:m10:materialize -- --bundle docs/assets/demo/v1.2.4/raw/m10-v124-demo-bundle.json
```

Materialize canonical demo assets from split raw captures (fallback):

```bash
pnpm demo:m10:materialize -- --v130-evidence docs/assets/demo/v1.2.4/raw/m10-v130-evidence-capture.json --digest-bundle docs/assets/demo/v1.2.4/raw/m10-digest-event-bundle.json
```

## Demo Storyboard

Use short segments (10-25s each) and keep one message per segment:
1. trust controls UI interaction:
: open settings, switch attack-mode profile, show deterministic counters.
2. CP3/CP4/v130 evidence run:
: run capture helpers in console and show pass verdict fields (`pass`, `failedChecks`).
3. digest correlation:
: show event chain in `getCrossDeviceSyncDigest(400)` and matching summary counters.

## Acceptance Gate for This Matrix

All items must be true before calling demo assets complete:
1. every required JSON file exists under `docs/assets/demo/v1.2.4/`,
2. every pass-lane file has gate `pass === true`,
3. digest summary includes non-null `summary.m10TrustControls`,
4. event chain includes at least one event for:
: `cp2_stability_gate`, `cp3_readiness_gate`, `cp3_suite_gate`, `cp4_closeout_gate`, `v130_closeout_gate`, `v130_evidence_gate`,
5. storyboard references only canonical helper APIs (no ad-hoc mutation scripts).

Track progress signal from status report:
1. `strictReady === true` means pass-lane captures are complete for this matrix.
