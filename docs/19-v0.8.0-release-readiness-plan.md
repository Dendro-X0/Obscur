# v0.8.0 Release Readiness Plan

_Last reviewed: 2026-03-03 (baseline commit 7f57b32)._

This document is the execution tracker for pre-`v0.8.0` hardening.

## 1) Scope Lock

1. Include only P0 hardening work:
   security assurance, messaging reliability, usability clarity, performance gates, release discipline.
2. Defer protocol redesign, major persistence migrations, and worker/off-main-thread pipeline redesign.
3. Enter bugfix-only code freeze on 2026-03-14.

## 2) Ticketized Worklist

### A. Security Assurance

1. `V080-SEC-001` Threat model complete and published.
2. `V080-SEC-002` Dependency audit run and triaged.
3. `V080-SEC-003` Secure defaults validated for relay/provider/user privacy settings.
4. `V080-SEC-004` Crypto-sensitive review checklist applied to all messaging/security PRs.

### B. Messaging Reliability

1. `V080-REL-001` Reduce relay-scope mismatch log spam and map to actionable UI state.
2. `V080-REL-002` Harden group join/leave retry/recovery behavior.
3. `V080-REL-003` Idempotency protection for message/event new-update-delete race windows.
4. `V080-REL-004` Burst + mixed-event regression tests for DM and groups.

### C. Usability and Ease of Use

1. `V080-UX-001` User-facing error copy pass for relay/sync/upload failures.
2. `V080-UX-002` First-run guidance for key ownership, backup, and recovery.
3. `V080-UX-003` Basic/advanced relay configuration UX split.
4. `V080-UX-004` Light/dark contrast verification for all updated messaging states.

### D. Performance Gates

1. `V080-PERF-001` Run synthetic load generator with seeded history + burst events.
2. `V080-PERF-002` Low-end profile performance pass with media-heavy timelines.
3. `V080-PERF-003` Confirm no duplicate/lost/out-of-order messages under burst.
4. `V080-PERF-004` Confirm p95 update latency and FPS median are within accepted range.

### E. Release Discipline

1. `V080-OPS-001` Release checklist evidence captured.
2. `V080-OPS-002` Changelog + docs synchronized to shipped behavior.
3. `V080-OPS-003` RC smoke pass (PWA + desktop).
4. `V080-OPS-004` Go/no-go review with explicit rollback decision.

## 5) Required Evidence Per Ticket

1. Code references (changed files).
2. Command output summary (tests/checks executed).
3. Risk note (what can still fail).
4. Rollback note (how to revert behavior safely).

## 6) v0.8.0 Go/No-Go Criteria

1. All `SEC-*`, `REL-*`, `UX-*`, `PERF-*`, and `OPS-*` tickets complete.
2. No open critical security findings.
3. No known high-severity messaging reliability regressions.
4. `pnpm version:check` passes and release-tracked manifests are aligned.
5. `pnpm docs:check` passes and release docs are current.
