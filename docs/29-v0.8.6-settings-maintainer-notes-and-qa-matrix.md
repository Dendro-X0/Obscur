# v0.8.6 Settings Maintainer Notes + QA Matrix

_Last reviewed: 2026-03-06 (baseline commit a6f71d2)._

## 1) Scope

This document closes:

1. `V086-DOC-001` Setup/config maintainer notes.
2. `V086-OPS-001` v0.8.6 targeted QA matrix for settings reliability.

## 2) Behavior Contracts

### Settings action status contract

All touched settings operations should surface deterministic phase feedback via `SettingsActionStatus`:

1. `idle`
2. `waiting`
3. `preparing`
4. `working` / `publishing`
5. `success`
6. `error`

### Progressive disclosure contract

1. Critical settings remain directly accessible.
2. Deep diagnostics should be collapsed by default (for example Identity Diagnostics).

### Runtime resilience contract

1. Expected runtime limitations must not raise red-overlay class errors.
2. Failure paths should degrade gracefully and return actionable guidance.

## 3) Troubleshooting Notes

### Profile publish stuck on waiting relays

1. Confirm at least one relay is enabled in Settings > Relays.
2. Confirm relay status is not `unavailable`.
3. Retry publish after reconnect window; relay flaps are retried with bounded backoff.

### Native folder open fails in Storage

Symptoms:

1. Open-folder action fails with shell scope/capability restrictions.

Expected behavior:

1. No crash/red overlay.
2. Action fails gracefully with user-facing message.
3. Bounded degraded log event is emitted.

### Notification test action does nothing

1. Check permission state in Notifications panel.
2. Ensure channel `Invites and system alerts` is enabled for test action path.

### Identity mismatch state

1. Identity panel shows mismatch diagnostics.
2. Key reveal flow must remain blocked until mismatch is resolved.

## 4) v0.8.6 Targeted QA Matrix

### A. Profile

1. Edit valid profile fields, publish, verify phase transitions and relay summary.
2. Enter invalid username/NIP-05/avatar URL and verify preflight blocking.
3. Simulate transient relay disconnect and verify bounded retry behavior.

### B. Appearance

1. Change language/theme and verify immediate UI reflection.
2. Toggle text scale/reduced motion/contrast assist and verify persistence after reload.
3. Use reset actions and verify deterministic success state.

### C. Notifications

1. Toggle each channel independently (`DM`, `mentions/replies`, `invites/system`).
2. Verify permission diagnostics display (`granted|denied|default|unsupported`).
3. Run test notification action and verify local delivery path.

### D. Identity

1. Verify identity overview (`hex`, `npub`, storage mode badge) renders.
2. Verify key reveal requires challenge and auto-hides after ~20 seconds.
3. Verify diagnostics are collapsed by default and expandable.
4. Verify mismatch state blocks unsafe key reveal.

### E. Security

1. Verify posture strip updates with toggles (`strong|moderate|weak` behavior).
2. Verify capability labels for clipboard/biometric/Tor.
3. Verify `Lock now` transitions session to locked state immediately.
4. Verify clear-local-data action requires confirmation.

### F. Relay + Storage polish

1. Relay quick-health strip shows connected count/latency/recommendation.
2. Preset apply works (`Default Stable`, `High Redundancy`, `Low Latency`).
3. Storage effective mode reflects NIP-96/local-vault toggles.
4. Storage metrics card updates after cache changes.
5. NIP-96 URL validation and manual provider check remain non-blocking.

### G. Moderation + Updates

1. Blocklist add/search/copy/unblock/unblock-all actions show deterministic status.
2. Privacy DM policy toggles persist (`everyone|contacts-only`).
3. Updates panel handles `dev` builds without false “update available”.
