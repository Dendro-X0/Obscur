# v0.8.7 Reliability Core Roadmap

_Last reviewed: 2026-03-06 (baseline commit 51a511b)._

## Release Theme

1. Reliability-first stabilization bridge before v0.9.0 feature expansion.
2. Adaptive relay behavior with deterministic fallback.
3. Sync correctness and storage resilience with local diagnostics.

## Locked Decisions

1. Relay policy: adaptive scoring + quorum publishing.
2. Storage scope: resilience and diagnostics only.
3. Rollout: feature-flagged (`reliabilityCoreV087`) default on.
4. Telemetry: local-only diagnostics counters.
5. No protocol/API/schema migration.

## Ticket Groups

1. `V087-RELAY-*`
   - Relay scoring model from success/latency/churn signals.
   - Quorum-aware publish contract with partial-failure reporting.
   - Jittered reconnect + duplicate reconnect suppression.
2. `V087-SYNC-*`
   - Deterministic checkpoint model per timeline.
   - Gap detection and targeted backfill request shaping.
   - Idempotent sync flow under relay flaps.
3. `V087-STORAGE-*`
   - Startup and on-demand storage health checks.
   - Local index integrity repair path (non-destructive).
   - Storage resilience counters for diagnostics.
4. `V087-UX/OPS-*`
   - Reliability status strip in Settings.
   - Standardized message action failure reasons.
   - Release preflight matrix check for required artifact rules.

## Acceptance Gates

1. No duplicate/lost/out-of-order regressions in DM/group burst scenarios.
2. Reduced reconnect thrash (suppressed duplicate reconnect scheduling).
3. Stable UI latency baseline (no regression against existing p95 target).
4. Storage health + repair path validated.
5. Release preflight passes including artifact matrix verification.
