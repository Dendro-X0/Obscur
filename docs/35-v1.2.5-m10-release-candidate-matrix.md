# 35 v1.2.5 M10 Release Candidate Matrix

_Last reviewed: 2026-03-25 (baseline commit 8e0ece2)._

This matrix defines the release-candidate evidence bundle while `v1.2.5` is the active M10 closeout-prep lane.

Goal:
1. reduce multi-command replay drift before `v1.3.0`,
2. keep capture and verification on canonical M10 bridge owner APIs,
3. produce deterministic strict-ready artifacts for closeout handoff.

## Output Location

Store all generated files in:
- `docs/assets/demo/v1.2.5/`

Automation:
1. initialize missing template files:
: `pnpm demo:m10:rc:init`
2. structure-only verification:
: `pnpm demo:m10:rc:check:structure`
3. materialize canonical pass-lane files from one-shot capture:
: `pnpm demo:m10:rc:materialize -- --capture docs/assets/demo/v1.2.5/raw/m10-v130-release-candidate-capture.json`
4. strict verification:
: `pnpm demo:m10:rc:check`
5. status report export:
: `pnpm demo:m10:rc:status`
: output: `docs/assets/demo/v1.2.5/m10-status.json`

## Required Release-Candidate Assets

1. `m10-v130-release-candidate-pass.json`
: release-candidate gate payload (`releaseCandidateGate.pass`).
2. `m10-v130-release-candidate-capture.json`
: full one-shot capture from canonical helper.
3. `m10-digest-summary.json`
: compact digest projection with anti-abuse + responsiveness + trust controls summary.
4. `m10-event-slices.json`
: CP2/CP3/CP4/v130 event chain slices.
5. `m10-demo-storyboard.md`
: concise operator replay script.

## Console Capture Command

Start with a clean event window:

```js
window.obscurAppEvents?.clear?.();
```

Capture the one-shot payload:

```js
copy(window.obscurM10TrustControls?.runV130ReleaseCandidateCaptureJson?.({ eventWindowSize: 400, expectedStable: true }));
```

Save the copied JSON string to:
1. `docs/assets/demo/v1.2.5/raw/m10-v130-release-candidate-capture.json`.

Materialize canonical artifacts:

```bash
pnpm demo:m10:rc:materialize -- --capture docs/assets/demo/v1.2.5/raw/m10-v130-release-candidate-capture.json
```

## Acceptance Gate for This Matrix

All items must be true before declaring `v1.2.5` release-candidate assets complete:
1. all required JSON files exist under `docs/assets/demo/v1.2.5/`,
2. strict gate booleans are `true` for:
: `cp2`, `cp3 readiness`, `cp3 suite`, `cp4 closeout`, `v130 closeout`, `v130 evidence`, and `releaseCandidateGate`,
3. digest summary includes non-null `summary.m10TrustControls`,
4. event chain includes at least one event for:
: `cp2`, `cp3Readiness`, `cp3Suite`, `cp4Closeout`, `v130Closeout`, `v130Evidence`, `v130ReleaseCandidate`,
5. storyboard references canonical helper APIs only.

Progress signal:
1. `strictReady === true` in `docs/assets/demo/v1.2.5/m10-status.json`.
