# v0.8.9 Stability + Release Integrity Roadmap

_Last reviewed: 2026-03-06 (baseline commit ab08104)._

## Summary

v0.8.9 is a stabilization-only bridge release before v0.9.x feature expansion.

1. Strengthen release determinism and post-tag verification.
2. Complete native/web boundary hardening for shared PWA surfaces.
3. Improve relay/sync/storage resilience and diagnostics under churn.

## Locked Decisions

1. Focus: Stability + Release (P0 only).
2. Production web policy: blocked by default (explicit env override only).
3. Release QA gate: targeted deterministic pack (not full-suite block).
4. No protocol/API/schema migrations in v0.8.9.

## Ticket Groups

1. `V089-REL-*`
   - Preflight gate expansion: clean tree + remote tag guard + local tag/HEAD guard.
   - CI signal contract checks.
   - Post-tag release verification script (artifacts + changelog/version/tag consistency).
2. `V089-ARCH-*`
   - Adapter-only native command usage in shared PWA settings/runtime paths.
   - Suppress expected unsupported-runtime noise with bounded logging.
   - Clarify production web policy messaging and override key.
3. `V089-RELAY/SYNC/STORAGE-*`
   - Relay circuit-state classification (`healthy | degraded | cooling_down`).
   - Checkpoint repair contract before backfill decision.
   - Storage recovery report expansion and write-retry counters.
4. `V089-QA-*`
   - Deterministic release test pack script.
   - Known-failure registry for non-blocking full-suite failures.

## Acceptance Gates

1. `pnpm release:test-pack` passes end-to-end.
2. `pnpm release:preflight` passes with CI contract checks.
3. Runtime web harness has no repetitive native-command error spam.
4. Reliability diagnostics expose relay cooling-down, checkpoint repair, and storage retry counters.
