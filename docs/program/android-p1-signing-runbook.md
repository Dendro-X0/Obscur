# Android P1 — build, signing, and smoke (Lane P)

**Cadence:** Pipeline is done for day-to-day development. **Functional smoke and UI/UX validation run at final wrap-up**, not in parallel with desktop feature work. See [stability-first-delivery.md](./stability-first-delivery.md).

**Status:** Active (v1.8.x Lane P)  
**Policy:** [mobile-desktop-version-policy.md](./mobile-desktop-version-policy.md) — no Play/App Store purchased certificates required for program testing.  
**Related:** [mobile-ui-stack-evaluation.md](./mobile-ui-stack-evaluation.md), [obscur-2.0-milestone-roadmap.md](./obscur-2.0-milestone-roadmap.md) (P1 row)

---

## Goal

Produce an **installable APK** from the same monorepo kernel as desktop (`TAURI_SHELL_TARGET=mobile`), signed with a **maintainer-owned** keystore (debug or release), and smoke on **emulator + one physical device** when hardware is available.

Desktop G6 online gates do **not** block this lane.

---

## Prerequisites

| Tool | Notes |
|------|--------|
| **JDK 17–24** | For the Gradle daemon — **not** JDK 25+; Android Studio bundled JBR is fine; set `JAVA_HOME` |
| **Android SDK** | `ANDROID_SDK_ROOT` or `ANDROID_HOME`; platform **android-36**, build-tools, **NDK** (side-by-side) |
| **Rust stable** | `rustup target add aarch64-linux-android armv7-linux-androideabi i686-linux-android x86_64-linux-android` |
| **Node 20+**, **pnpm** | Same as desktop |
| **Android Studio** | Optional but recommended for SDK install + emulator |

Automated check (non-destructive):

```bash
pnpm verify:android-prerequisites
```

## Version alignment

Before any release-tracked Android build:

```bash
pnpm version:sync
```

Writes Android `tauri.properties` under `apps/desktop/src-tauri/gen/android/app/` (`versionName` / `versionCode`) from root `package.json`. See [mobile-desktop-version-policy.md](./mobile-desktop-version-policy.md).

## Debug APK (canonical first smoke)

```bash
pnpm verify:android-prerequisites
pnpm build:android:debug
```

**Emulator-only (faster — one ABI, ~4× less Rust compile):**

```bash
pnpm build:android:debug:emulator
```

Use this for day-to-day smoke after client fixes. Reserve full universal debug/release for pre-release.

See [stability-first-delivery.md](./stability-first-delivery.md) — do not loop `build:android:release` while post-login crashes are open.

Uses the **debug keystore** generated by Android Gradle — no secrets in repo.

Equivalent (manual):

```bash
pnpm version:sync
pnpm verify:android-prerequisites
cross-env TAURI_SHELL_TARGET=mobile pnpm -C apps/desktop tauri android build --apk --debug
```

**Output (typical):**  
apps/desktop/src-tauri/gen/android/app/build/outputs/apk/universal/debug/app-universal-debug.apk

### Install on emulator or device

```bash
adb devices
adb install -r apps/desktop/src-tauri/gen/android/app/build/outputs/apk/universal/debug/app-universal-debug.apk
```

### P1 smoke checklist (manual)

| # | Check |
|---|--------|
| 1 | Cold start — app opens without immediate crash |
| 2 | Auth screen loads (offline / online per build flags) |
| 3 | Unlock or create identity — no infinite spinner |
| 4 | Navigate Chats / Settings — tab bar + back behave |
| 5 | Optional online: relay list loads when `NEXT_PUBLIC_OBSCUR_EXPERIMENT_ONLINE=1` in mobile shell build |

Record Pass/Fail in maintainer notes; full community matrix stays frozen until G6-4 desktop coordination sign-off.

**v1.8.12 wrap-up helper:**

```bash
pnpm p12:android-smoke -- --build --wait-device=180
```

Prints a maintainer record block for [v1.8.12 demo matrix](../assets/demo/v1.8.12/README.md).

## Release APK / AAB (local / decentralized signing)

**Never commit** keystores, passwords, or `key.properties` to git (see `apps/desktop/src-tauri/gen/android/.gitignore`).

### 1. Generate a release keystore (once per maintainer machine)

```bash
keytool -genkeypair -v \
  -keystore obscur-release.jks \
  -alias obscur \
  -keyalg RSA -keysize 2048 -validity 10000 \
  -storetype PKCS12
```

Store `obscur-release.jks` **outside** the repo (e.g. password manager + encrypted backup).

### 2. Build signed release artifacts

From repo root, with env vars (no file committed):

```bash
export TAURI_ANDROID_KEYSTORE_PATH="/absolute/path/to/obscur-release.jks"
export TAURI_ANDROID_KEYSTORE_PASSWORD="…"
export TAURI_ANDROID_KEY_ALIAS="obscur"
export TAURI_ANDROID_KEY_PASSWORD="…"

pnpm build:android:release
```

Produces universal **APK** and **AAB** under `gen/android/app/build/outputs/`.

### 3. Verify signature

```bash
# Android SDK build-tools apksigner (adjust version dir)
apksigner verify --verbose apps/desktop/src-tauri/gen/android/app/build/outputs/apk/universal/release/*.apk
```

Optional fingerprint audit:

```bash
keytool -list -v -keystore /path/to/obscur-release.jks -alias obscur
```

## CI / GitHub Actions (optional)

Reference workflow (archived v1.3.15): decode `ANDROID_KEYSTORE_BASE64` secret to a local release keystore (gitignored), then:

```bash
pnpm -C apps/desktop tauri android build --apk --aab --ci
```

Play Console upload is **out of scope** for indie/decentralized distribution; sideload / Obtainium / F-Droid are the intended channels.

## Mobile shell vs desktop shell

| Env | Effect |
|-----|--------|
| `TAURI_SHELL_TARGET=mobile` | `NEXT_PUBLIC_MOBILE_SHELL=1` during `build:pwa-shell` |
| `TAURI_SHELL_TARGET=desktop` | Default desktop chrome |

Android builds must use **mobile** so tab bar / safe-area policies match the product shell.

## Troubleshooting

| Symptom | Action |
|---------|--------|
| NDK not found | SDK Manager → NDK (Side by side); set `ANDROID_NDK_HOME` |
| `compileSdk 36` missing | Install Android 16 / API 36 platform |
| Rust link errors | `rustup target add` for all four Android triples |
| Cleartext HTTP blocked in release | Debug manifest allows cleartext; release uses HTTPS only — use TLS relays / coordination |
| Version mismatch on tag | Run `pnpm version:check` and `pnpm version:sync` |
