## Stability vs features (no ambiguity)

1. **P0 shell crash** (fatal `RootErrorBoundary`, max update depth, hooks mismatch) — fix **before** new feature rows in the same concentration unit; record in issues register.
2. Fixes must follow **one owner per loop** ([ui-effect-stability-policy.md](./ui-effect-stability-policy.md)) — not ad-hoc provider patches without owner doc update.
3. **`pnpm verify:stability`** and relay/runtime Vitest — required on stability-touching PRs; not a substitute for Phase B matrix rows.
4. Manual verification: deferred during Phase A batch work is OK; **required at Phase B exit** per [manual-verification-environment.md](./manual-verification-environment.md) unless row is **Blocked env**.

## Resume checklist (maintainer — when restarting v1.9.x)

1. Complete [design-goals-and-constraints.md](./design-goals-and-constraints.md) § pre-implementation gate.
2. Read this contract + [current-session.md](../handoffs/current-session.md).
2. Set handoff: `Session Status: Active — v1.9.4 Phase A` (or correct unit/phase).
3. Confirm active scope doc matches train (default: **v1.9.4-scope.md**).
4. Pick **one** next atomic step from scope doc or register P0 — update handoff **before** coding.
5. Run `pnpm release:test-pack -- --skip-preflight` once to confirm baseline.
6. **Do not** open superseded charter or stale roadmap files for todos.
