## Goal

Ship **basic anti-fraud and anti-bot modules** plus an **internal security validation pass** so v2.0.0 can honestly claim a **high-trust communication stack** for B2Pro audiences:

| Pillar | Meaning for Obscur |
|--------|-------------------|
| **E2EE** | DM and sealed group ciphertext; decrypt only on recipient device |
| **No data uploading** | No vendor telemetry, no server-side plaintext NLP, no central user graph — assessments run **recipient-local** |
| **Attack resistance** | Profile isolation, transport/gateway boundaries, relay operator hardening when configured via Obscur |
| **Abuse assistance** | Recipient-local fraud signals + steward-controlled bot gates — **help users decide**, not platform enforcement |

**Marketing honesty:** v1.9.5 delivers a **baseline trust band**, not a finished “AI anti-scam suite.” v2.0 gate doc must list remaining **A** rows and rule-pack expansion as post-v2.1 unless re-chartered.

---

## Product position (maintainer intent)

Obscur targets **technical / B2Pro users** who deploy **private-trust infrastructure** (coordination + operator relay). Anti-fraud is **recipient-sovereign**: warnings on Party B’s device only; sender never notified; no account bans by Obscur.

Canonical architecture: assessments attach to **dm-kernel first** ([workspace-kernel-manifest.md](./workspace-kernel-manifest.md)); community bot defense extends existing **B2 inbound** steward gates.

## In scope — Phase B (verification)

Run desktop A/B ([manual-verification-environment.md](./manual-verification-environment.md)) plus automated gates:

| Matrix ID | Scenario | Pass criteria |
|-----------|----------|---------------|
| TRUST-1 | Cold DM + financial mention | Recipient sees **elevated** banner; sender unaware; message delivered |
| TRUST-2 | Dismiss / snooze | Same thread suppresses repeat banner for cooldown window |
| TRUST-3 | Trusted peer thread | No banner on established contact with benign financial mention (low false positive) |
| BOT-1 | B2 keyword flood | Bot rate-limited; group still usable; steward can disable |
| BOT-2 | Non-allowlisted bot pubkey | Trigger ignored; honest UI |
| SEC-R1 | Managed workspace create | Blocked or degraded on `public_default` relay without coordination |
| SEC-V* | Internal checklist | All § in security validation doc **Pass** or **Accepted** with copy |

Update [unified-verification-issues-register.md](./unified-verification-issues-register.md) before Phase C.
