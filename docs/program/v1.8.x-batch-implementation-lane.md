---

## Why this lane exists

Release CI and per-patch manual matrices were slowing delivery without improving day-to-day code velocity. This lane separates:

| Phase | When | Gate |
|-------|------|------|
| **Implement** | Now | Automated tests + `pnpm release:test-pack` on push (reliability-gates) |
| **Verify** | When maintainer has time / environment | [deferred-manual-verification-checklist.md](./deferred-manual-verification-checklist.md) |
| **Tag / publish** | After batch + optional manual pass | One tag per **user-visible** milestone — not per CI tweak |

Per [version-line-policy.md](./version-line-policy.md): **do not bump version for CI-only fixes.**

## Daily commands (implementation mode)

```bash
pnpm dev:desktop:online          # primary desktop iteration
pnpm dev:mobile-shell:online     # mobile shell without APK
pnpm -C apps/pwa exec vitest run <target>   # focused tests
pnpm release:test-pack           # pre-push confidence (not Full Release)
```

**Do not** require green Full Release or manual matrix before merging the next wave item.

**Do not** pause implementation for incremental manual testing (e.g. K-M1 after B2, rapid-nav after each N lane). Batch manual rows when a milestone is code-complete or before a user-visible tag.

---

## Batch exit (when to tag)

Tag when **any one** of these is true — maintainer choice:

1. A **named wave** is complete (e.g. P13 + B2) and automated gates green.
2. You want users to download a build — publish once, not per fix.
3. Deferred checklist has been partially or fully exercised and regressions filed.

**Do not** tag solely because CI workflow changed.
