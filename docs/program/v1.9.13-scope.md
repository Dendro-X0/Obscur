## Why this unit (product truth)

| Claim we want for public | Owner in 1.9.13 |
|--------------------------|-----------------|
| Attachments saved locally are **ciphertext at rest** | Rust `libobscur::les` + Tauri LES commands |
| Chat Save and Secure Upload share **one catalog** | LES list/read/commit ‚Äî not legacy `local-media-store` |
| Preview works after refresh / reopen | `useLesVaultMedia` + `VaultMediaGrid` (R6) |
| Monitor on the wire does not see Vault plaintext | LES never uploads vault blobs; chat E2EE separate |

Kernel thesis: **client owns encryption**; transports remain adapters.
