# v1.5.x feature roadmap (product focus)

**Status:** Active — **features over distribution**  
**Last updated:** 2026-05-18  
**Distribution policy:** Installers/APK/F-Droid/iOS are **optional**; tags when convenient, not a sprint blocker.  
**Related:** [strategic-direction.md](./strategic-direction.md), [mobile-desktop-version-policy.md](./mobile-desktop-version-policy.md)

---

## How to read this plan

| Label | Meaning |
|-------|---------|
| **Refine** | Make existing flows faster, clearer, or more reliable — no new user-facing capability |
| **New** | New surface or behavior users did not have before |
| **Deferred** | Explicitly not v1.5.x |
| **Distro** | Signing, stores, F-Droid, AltStore — **shelved** until voluntarily picked up |

**Current emphasis (v1.5.5+ dev on `main`):** ~**85% refine / 15% small new** (mobile layout slices only when they reuse shared kernel).

## What “focus now” means for implementation

1. Pick **one** item from **Lane R** (reliability).  
2. Add **tests** in `pnpm release:test-pack`.  
3. Only then pick **U** or **M1**.  
4. **Do not** block merges on: NPM tokens, Play Console, Apple ID, F-Droid metadata.

---

## Agent / maintainer default

- **Yes:** shared kernel fixes in `apps/pwa` / `packages/*`.  
- **No:** new parallel DM send path; new global “current user”; release infra unless asked.  
- **Distro:** document in [indie-distribution.md](./indie-distribution.md) when resumed (optional future doc).
