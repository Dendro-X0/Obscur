## Theme

Improve **network connection pooling** and ship **user-facing transport presets** so people can pick convenient defaults (public relays, private mesh, Tor-required) without becoming Nostr experts — while the **client always owns encryption** (E2EE envelopes + LES ciphertext). Adapters are configurable; plaintext never belongs to the network.

This unit is **transport UX + pool reliability**, not community roster repair.

---

## Product bar

| Must be true | Explicitly not required |
|--------------|-------------------------|
| Client encrypts messages and local vault data | Perfect anonymity / traffic analysis resistance |
| Pool health is readable; failover among configured endpoints | Every dialect L3-proven on one host |
| Presets + manual URL add/edit without code changes | Community roster parity (ACC-02) |
| Wire/monitor cannot read **plaintext** | Signed store listing / `v2.0.0` tag |

**One sentence for users:** “Obscur encrypts your data on the device; you choose which networks carry the ciphertext.”
