# v1.5.5 — Mobile production (planned)

**Status:** Planning  
**Predecessor:** [v1.5.4 release](../releases/v1.5.4-release.md) (desktop production)  
**Policy:** [mobile-desktop-version-policy.md](./mobile-desktop-version-policy.md)

---

## Goal

Ship **mobile as a production lane** on the **same `1.5.x` version line** as desktop — not a forked app or separate semver.

**Bar for success:**

- Signed Android APK (release keystore in CI).
- Device matrix M1–M3 green on maintainer hardware.
- `versionName` / in-app version matches desktop for the same tag.
- Incremental native work (push, background, shell UX) without duplicating kernel logic in Kotlin/Swift.

---

## Non-goals

- Separate “Obscur Mobile 2.0” versioning.
- Full tab-by-tab mobile UI rewrite in one release.
- iOS App Store ship unless signing + scope explicitly opened.

---

## Depends on v1.5.4

- Unified icons (`pnpm icons:regenerate`).
- `version:sync` Android metadata path.
- Shared PWA mobile shell build (`TAURI_SHELL_TARGET=mobile`).
