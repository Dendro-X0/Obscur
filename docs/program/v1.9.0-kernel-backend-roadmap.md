## Why this band exists

v1.8.x (Lane T) fixes **symptoms** on a Nostr-shaped stack: leave outbox, roster splits, honest copy. Users still hit **structural** failures:

- Leave on A does not converge on B without fragile gossip.
- “Relay-confirmed” implies server truth on public relays.
- Membership, messaging, and roster logic live in React hooks and parallel Nostr kind handlers.
- `docs/future/` and [protocol 22](../protocols/22-local-first-decentralized-protocol-architecture.md) describe the right architecture but **do not gate releases**.

Without **v1.9.x**, Obscur remains an **unfinished Nostr client** — features exist, but **community and cross-client membership are not reliably usable** on sovereign + public relay setups.

**Lane K goal:** Make the **kernel + coordination backend** the source of truth; relegate Nostr to **transport**. Ship **usable** leave/re-invite, membership fanout (opt-in), and one read model per domain before v2.0 platform work.
