**Status:** Shipped (tag `v1.5.2`)  
**Program lane:** Application shell, community surfaces, relay transport  
**Strategic frame:** [strategic-direction.md](./strategic-direction.md) — decent Nostr client; no kernel rewrite  
**Predecessor:** [v1.5.1 release](../releases/v1.5.1-release.md)  
**User-facing release notes:** [v1.5.2-release.md](../releases/v1.5.2-release.md)  
**Gate checklist:** [v1.5.2-ui-performance-gate.md](../releases/v1.5.2-ui-performance-gate.md)

---

## Goal

Ship **v1.5.2** as a **feel-fast** patch: first navigation and in-app tab switches should feel immediate; relay work should not compete with UI on the main thread or flood the console during normal use.

**Bar for success:** a maintainer can switch Chats → Network → Vault → Settings and Network tabs (Contacts ↔ Communities) without multi-second blank/skeleton stalls; relay standby probing and pool churn are bounded and deferrable.

---

## Non-goals

| Non-goal | Rationale |
|----------|-----------|
| Kernel / protocol work from `docs/future/` | Not gating v1.5.x |
| Cooperative recall / delete-for-everyone UI | v1.6 lane |
| New product features (DM, groups, vault semantics) | Perf/UX only |
| “Fix all relays worldwide” | Scope to connection policy, deferral, and diagnostics |
| Marketing claims about global sync speed | Evidence-based gates only |

## Workstreams

### WS-A — Route & shell performance

**Owner:** `app-shell.tsx`, `route-navigation-warmup.ts`, `navigation-prefetch-warmup-policy.ts`, page transition recovery.

| Task | Done when |
|------|-----------|
| Sidebar hover/focus prefetch for `/network`, `/vault`, `/search`, `/settings` | First click does not pay full dynamic-import latency |
| Idle warmup after shell mount (desktop + web) | Policy does not skip desktop incorrectly |
| Shorten or gate route transition overlay | No 300–700ms artificial delay on every navigation |
| Regression tests in `release:test-pack` | `navigation-prefetch-*`, `app-shell.test.tsx` green |

### WS-B — Network & community surfaces

**Owner:** `network-dashboard.tsx`, `group-home-page-client.tsx`, community confirmation routes.

| Task | Done when |
|------|-----------|
| Tab switches use `startTransition`; animation ≤150ms or `prefers-reduced-motion` | Contacts ↔ Communities feels instant |
| Preload `group-home-page-client` when Network warms | Opening a community is not a cold chunk |
| Dedicated confirm pages for leave/block/purge (no `ConfirmDialog` on community page) | Manual: no invisible overlay |
| Optional: defer GroupDiscovery / heavy lists until tab visible | Network “all” tab does not block first paint |

### WS-C — Relay efficiency & noise

**Owner:** `relay-provider`, `use-standby-latency-probe`, `enhanced-relay-pool`, `native-relay`.

| Task | Done when |
|------|-----------|
| Defer standby latency probes until after first paint + `document.visibilityState === visible` | Startup does not open N× parallel probe sockets immediately |
| Cap concurrent probe / connect attempts | Configurable budget (e.g. 2–3 at a time) |
| Reduce dev-console WebSocket warning spam | Probes close cleanly; failures logged via `logAppEvent` at debug, not browser warn storm |
| Primary pool: avoid duplicate connects to same URL | One active connection per scoped relay where possible |
| Settings/relay UI still shows latency when probed | No regression on relay health display |

### WS-D — Heavy page cold start (secondary)

**Owner:** `vault-page-client`, `settings-page-client`, `search-page-client`.

| Task | Done when |
|------|-----------|
| Vault grid: show lightweight shell first; load media index async | Skeleton does not block entire shell for seconds |
| Settings: defer non-critical panels until section opened | First open acceptable |
