## BOT-1 — B2 keyword flood + steward disable

**Maps to:** SEC-B1/B3/B4 · checklist V5-3, V5-4 · [community-inbound-bot.md](../messaging/community-inbound-bot.md)

**Requires:** Managed workspace with 2+ members, operator relay, bot key registered.

### Setup

1. Create managed workspace (Tester1 steward); invite Tester2.
2. Generate bot key: `pnpm community-bot:generate-key -- --nsec`
3. Register bot pubkey in **Manage → General → Outbound bots**.
4. Add inbound keyword trigger: keyword `help`, reply `pong`, enabled.
5. Export room key / group id; start runner:

```bash
OBSCUR_BOT_NSEC=nsec1… \
OBSCUR_BOT_RELAY_URL=ws://127.0.0.1:7000 \
OBSCUR_BOT_GROUP_ID=<group-id> \
OBSCUR_BOT_ROOM_KEY_HEX=<64-hex> \
OBSCUR_BOT_ALLOWED_PUBKEYS=<bot-pubkey-hex> \
pnpm community-inbound-bot
```

### Flood scenario

| Step | Action | Expected |
|------|--------|----------|
| 1 | Tester2: post 20+ messages containing `help` within ~1 minute | Group remains usable; chat not frozen |
| 2 | Observe bot replies | At most **6 replies/min** (default rate limit); excess matches skipped |
| 3 | Runner logs | Rate-limit skip messages; no relay melt / error storm |
| 4 | Tester1 steward: **Manage → General → Inbound triggers** — disable bot master switch | UI shows paused/disabled state |
| 5 | Tester2: post `help` again | **No** new bot replies |

**Pass:** Rate limit holds; steward disable stops replies; group usable throughout.
