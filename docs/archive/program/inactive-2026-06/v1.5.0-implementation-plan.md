// packages/runtime/src/profile-bus.ts
export function createProfileMessageBus(params: {
  profileId: ProfileId;
}): ProfileMessageBus {
  const handlers = new Set<EventHandler>();
  return {
    publish: (event) => handlers.forEach(h => h(event)),
    subscribe: (handler) => { handlers.add(handler); return () => handlers.delete(handler); }
  };
}

## Phase 2: Storage — Contract-First, Dual-Engine (Weeks 3-4)

### Objective
Establish **shared persistence contracts** in the monorepo (`@dweb/storage-contracts`) so **PWA**, **desktop**, and **mobile** can iterate behind the **same TypeScript ports** while using **different engines**:

| Surface | Engine | Notes |
|---------|--------|--------|
| **Desktop / Mobile (native)** | **SQLite** (+ Drizzle in TS and/or SQL migrations owned by **Rust** — pick **one** migration owner per build) | Converges with the long-term **single Rust backend**; TS may call Tauri commands or a local Drizzle client — both must not each run conflicting migrations. |
| **PWA / Web** | **IndexedDB** (via `@dweb/storage` and related app code) | **No Drizzle/SQLite in the browser** for production PWA; capability gaps (**Tor**, **multi-profile in one process**) are explicit, not implied. |

Phase 2 is **not** “one ORM everywhere”; it is **one contract, multiple adapters**, so later iterations (Rust-first writes, repository swaps) do not rewrite product logic.

### Architecture decision (contract-first, 2026-05)

1. **Ports before bulk migration** — Define narrow interfaces in `@dweb/storage-contracts`; implement **IndexedDB** and **SQLite** adapters incrementally; move call sites behind the port when a slice is ready.
2. **Single migration owner on native** — Choose **`rust`** (preferred toward a shared Rust backend) **or** `typescript-drizzle`; document in crate/app; never apply two independent migration streams to the same `app.db`.
3. **Parallel with Phase 1** — Profile bus / `getResolvedProfileId` / dual-publish work **continues** until grep gates are met; storage slices **must not** introduce a second owner for the same lifecycle (see `AGENTS.md`).
4. **Cross-surface testing** — **Desktop (A) + Chrome guest (B)** is **two clients**; correctness requires **relay evidence** and **per-client persistence**, not only same-process bus isolation.

### Shared contract package

New workspace package: **`@dweb/storage-contracts`** (`packages/dweb-storage-contracts`).

- **`migration-policy`** — `NativeMigrationOwner`, default policy toward Rust-owned migrations.
- **`runtime-capabilities`** — `ClientStorageCapabilities` (e.g. `torBundled`, `multiProfileSameProcess`, `sqlMaterialization`) for gating UI and tests.
- **`scoped-context`** — `ScopedPersistenceScope` (`profileId` + `publicKeyHex`) for all scoped reads/writes.

**Native DB owner (Phase 2):** Desktop SQLite schema and migrations are owned by **`libobscur`** (`packages/libobscur`); TypeScript does not apply parallel migrations to the same file. See **`@dweb/storage-contracts/migration-policy`** for the `NativeMigrationOwner` enum (default **`rust`**).

Repository interfaces (e.g. `MessageRepository`) **grow here** as methods are extracted from `apps/pwa`; avoid importing app types into this package — use generics or duplicate minimal DTOs until stable.

### Week 7: System Integration

// apps/desktop/src/app.tsx
function DesktopApp() {
  const [container, setContainer] = useState<DesktopContainer | null>(null);
  
  useEffect(() => {
    const profileId = getProfileFromWindowParams(); // Explicit, not global
    const container = createDesktopContainer({ profileId });
    setContainer(container);
    
    // Initialize relay pool with profile scope
    container.services.relay.connect();
    
    return () => {
      container.destroy(); // Cleanup subscriptions
    };
  }, []);
  
  if (!container) return <Loading />;
  
  return (
    <ContainerProvider value={container}>
      <App />
    </ContainerProvider>
  );
}
