## Goal

Ship **v1.5.3** as a **stay-smooth** patch on desktop/web **and** establish **mobile release foundations** so v1.5.4+ can ship a real mobile UI without forking the monorepo kernel.

**Bar for success:**

- Desktop/web: Vault cold path and at least one long-session stability fix are evidence-backed.
- Mobile: Android CI builds a **mobile-shell** static bundle; maintainer can install APK and pass M1 smoke matrix.
- No new product features; no v1.6 cooperative-recall UI.

---

## Non-goals

| Non-goal | Rationale |
|----------|-----------|
| Full mobile UI rewrite (all tabs) | v1.5.4+ incremental surfaces |
| React Native / Flutter greenfield app | See stack evaluation — deferred unless WebView blocks ship |
| v1.6 cooperative recall / delete-for-everyone UI | Separate lane |
| Kernel rewrite from `docs/future/` | Not gating v1.5.x |
| iOS App Store ship (if signing blocked) | Document matrix; CI may stay non-blocking |

**Owner:** `vault-media-aggregator.ts`, `use-vault-media.ts`, `search-page-client`, `settings-page-client`.

**Owner:** profile runtime, account projection, relay provider, sign-out path.

| Task | Done when |

| Task | Done when |
