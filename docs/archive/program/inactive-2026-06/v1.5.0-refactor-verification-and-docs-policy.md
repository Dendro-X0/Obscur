# v1.5.0 — Refactor verification and documentation policy

**Last updated:** 2026-05-14

## Purpose

Govern **how we verify** and **how we persist progress** while the v1.5.0 **architecture refactor and upgrade** track is open. This is product process, not a runtime contract.

## Client and manual testing (deferred)

Until the items below are satisfied, **do not use client-facing or manual A/B sessions** as a required gate for refactor completion:

1. **R1** and **R2** exit criteria in `docs/program/v1.5.0-architecture-refactor-queue.md` are met (single-owner / multiplicity outcomes as defined there).
2. **`docs/encyclopedia/12-core-architecture-truth-map.md`** is updated so the affected rows document the **actual single owner** (per the architecture change gate described in that doc).
3. Repository checks agreed for the slice are green — at minimum **`pnpm docs:check`**, **`pnpm exec tsc --noEmit`** from `apps/pwa`, and any **narrow contract tests** that lock a completed slice.

**During deferral:** use automated checks, focused tests, and existing diagnostics (`logAppEvent`, `window.obscurAppEvents` where applicable). Treat regressions as engineering evidence, not as a signal to widen manual testing before the refactor exit.

## Checkpoint → documentation (required)

After each **completed checkpoint** (owner or contract decision, landed slice, blocker change, or likely thread end — align with `AGENTS.md`):

1. **`docs/handoffs/current-session.md`** — refresh `Last Updated`, snapshot/evidence, `Next Atomic Step`, and append one checkpoint entry (or run `pnpm context:checkpoint -- --summary "..." --next "..."` when you use the continuity scripts).
2. **`docs/program/v1.5.0-refactor-checkpoints.md`** — append one **dated line** (UTC): summary + optional “next”. This file is the **durable chronological log** of refactor progress without chat history.
3. **`docs/program/v1.5.0-architecture-refactor-queue.md`** — if an R1–R4 slice materially advances or exits, bump **Last updated** and adjust the slice text.
4. Run **`pnpm docs:check`** on doc edits before merge.

**Anti-duplication:** keep long narrative in the handoff only; the checkpoint log stays short bullets; the queue stays ordered backlog and exit language.
