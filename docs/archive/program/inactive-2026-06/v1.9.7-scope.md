# v1.9.7 — Auth subtraction + ship prep

**Status:** Complete (2026-06-13)  
**Predecessor:** v1.9.6 (session persistence redesign, profile picker, password-only return login)  
**Related:** [v1.9.6-session-persistence-redesign.md](./v1.9.6-session-persistence-redesign.md) · [v1.9.x-execution-contract.md](./v1.9.x-execution-contract.md)

---

## Theme

**Subtract infeasible login assist; keep the wallet-style auth model that works.**

v1.9.6 landed meaningful auth UX: password-only unlock on return visits (no username field), Stay signed in consent, profile picker routing, Lock vs Log out semantics. Local login assist (OS keychain autofill / save prompts) is **withdrawn** — it duplicated wallet unlock without delivering reliable value.

---

## In scope (v1.9.7)

| Item | Outcome |
|------|---------|
| **Withdraw login assist** | Remove title-bar sign-in assist, save prompts, autofill, vault service, Tauri `login_assist_*` commands |
| **Keep password-only login** | Log In tab: device password only; username lives in Settings / identity |
| **Keep Stay signed in** | Native keychain session path unchanged (`init_native_session`) |
| **Bundle export cleanup** | Remove saved-login export toggle from encrypted workspace bundle |
| **Handoff + charter** | Mark LLA withdrawn; next atomic step points at v1.9.7 gate |

---

## Out of scope

- Cold-boot silent restore (AUTH-SESSION-1) — remains deferred per v1.9.6 feasibility gate
- Auth Assistant / browser-style credential managers
- Anti–social-engineering module (future track)
