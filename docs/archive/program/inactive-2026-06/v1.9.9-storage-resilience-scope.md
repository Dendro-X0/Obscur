# v1.9.9 — Storage resilience & ops hardening

**Status:** Active (2026-06-15)  
**Builds on:** [v1.9.8-portable-storage-and-encryption-charter.md](./v1.9.8-portable-storage-and-encryption-charter.md)  
**Predecessor exit:** v1.9.8 Phase 4 manual evidence + tag

---

## 1. Problem

Portable data roots on removable or slow media expose failure modes that block the whole client:

| Observed (USB soak) | Impact |
|---------------------|--------|
| Slow reads on old USB | Long splash; web bootstrap watchdog fires |
| Drive disconnected while junction points at `K:\` | Startup hangs on `fs::create_dir_all` / SQLite open |
| OS error 32 / Controlled folder access | Migration copy aborts mid-flight |
| Generic “Recover + Reload” copy | User cannot reach Storage or encrypted export recovery |

**Goal:** Obscur must **degrade gracefully** when the physical data root is slow or unreachable — never infinite hang; always offer **actionable recovery** (reconnect folder, import backup, AppData copy).

---

## 2. Threat / scenario model

| ID | Scenario | Target outcome |
|----|----------|----------------|
| **R1** | USB unplugged; junction still targets missing path | App opens within bounded time; recovery screen explains issue |
| **R2** | USB present but very slow | Boot completes; optional “slow storage” notice; no false “data loss” |
| **R3** | Migration interrupted (copy fail) | Source authoritative; partial destination discardable (v1.9.8 copy semantics) |
| **R4** | Migration blocked (CFA, error 32) | Actionable error before bind change |
| **R5** | User needs data while drive away | Encrypted `.obscur-bundle` export path still documented / reachable from recovery UI |

#### Phase C manual matrix (how to record)

1. Set data root to removable drive (e.g. `K:\\...\\app.obscur.desktop`) and confirm Obscur restarts.
2. Quit Obscur completely.
3. Unplug/eject the drive.
4. Launch Obscur and time until recovery UI appears.
   - Expected: **< 5 seconds**
   - Expected screen: **“Data folder unavailable”** with reconnect action
5. Reconnect the drive.
6. Click **Reconnect data folder** and pick the folder containing `profiles/` and `obscur.sqlite3`.
7. Confirm Obscur restarts and can unlock + load DMs/vault.

Record: drive model, path, time-to-recovery-ui, pass/fail, any OS prompts.

## 6. Exit criterion

Desktop native with custom data root on removable media:

1. Unplug drive → app shows recovery UI within **5 seconds** (no infinite splash).
2. Reconnect drive → user can **Reconnect folder** without reinstall.
3. Encrypted export path documented from recovery UI.
4. No regression to v1.9.8 copy-migrate semantics (source never deleted on failure).
