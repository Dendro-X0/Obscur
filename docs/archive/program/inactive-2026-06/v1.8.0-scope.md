# v1.8.0 — Open the v1.8.x program band

**Status:** **Next band opener** (after tag **v1.7.1**)  
**Not a feature release:** This tag marks the **minor** line change `1.7.x` → `1.8.x`. Feature work ships in **`v1.8.1+`** patches.  
**Program:** [obscur-2.0-milestone-roadmap.md](./obscur-2.0-milestone-roadmap.md)

---

## Why v1.8.0 exists

Semver convention for this repo:

| Step | Version | Meaning |
|------|---------|---------|
| Last **v1.7.x** patch | `v1.7.1` | Close Phase 3 stabilization band |
| **Minor** bump | `v1.8.0` | Start **v1.8.x** line (all pre-2.0 work before v2.0.0) |
| **Patch** bumps | `v1.8.1`, `v1.8.2`, … | One focused milestone per tag where possible |

Skipping **`v1.8.0`** was an error in an early roadmap draft (treating “use patch bands” as “start patches at .1”). **Correct policy: always tag `v1.8.0` when opening the band.**
