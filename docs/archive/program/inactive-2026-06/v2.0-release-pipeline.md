## Phase 1 — Features complete (implementation + testing)

**Purpose:** Close all **in-scope** product rows before any packaging, website, or demo polish.

### 1a — Implementation (I)

Every row in [version-roadmap-scope.md](./version-roadmap-scope.md) master checklist must be **✓ I**, **◐ → ✓**, or **—** with explicit **defer/accept** recorded (not silent).

| Lane | Phase 1 focus |
|------|----------------|
| **K** | K3–K5 verification backlog (coordination leave, sovereign copy, read models) |
| **C** | C-4.1, C-4.2; **C-4.3** accept or implement — record in register |
| **T** | REL-001–004, MEM-002–006, MED-001–002; **REL-005** close or accept |
| **X** | X1–X3 verify; X4–X6 remain **A** |
| **P** | P3a–d manual soaks; P3 gap list **closed or accepted** per [obscur-native-sqlite-policy.md](./obscur-native-sqlite-policy.md); P1 Android Tier 1 **Pass** or **blocked_env**; P2 **document defer** in register if not built |

**Not Phase 1:** website, demo GIFs, signed release channel polish, public promotion, DM-002–007, iOS.

**Automated gate (continuous):** `pnpm release:test-pack` green on the Phase 1 exit commit.

### 1b — Verification (V)

One **unified verification round** on the Phase 1 exit commit:

1. Run [unified-verification-matrix.md](./unified-verification-matrix.md) (desktop A/B where applicable).
2. Record outcomes in [unified-verification-issues-register.md](./unified-verification-issues-register.md).
3. No silent **Open** P0/P1 for in-scope rows; limitations use **accepted_limitation** with copy links.

**Manual rows that must Pass or be explicitly skipped/blocked:**

| ID | Source |
|----|--------|
| P3d restart soak | Groups + sealed thread survive desktop restart (two profiles) |
| K-M coordination | Leave → peer sees excluded; sovereign/managed copy honest |
| C-4.x | Mode-aware create + manage hub (v1.8.1 / v1.8.2 demo matrices) |
| Android Tier 1 | [android-p1-smoke-checklist.md](./android-p1-smoke-checklist.md) — debug APK OK if release signing waits for Phase 3 |

**Phase 1 exit:** [version-roadmap-scope.md](./version-roadmap-scope.md) § v2.0.0 exit **A + B + C** satisfied (features + platform truth + verification). Update scope doc **I/V columns** to final state.

**Optional:** tag `v1.9.4` as an engineering checkpoint **only after** Phase 1 exit — not required for the pipeline.

## Phase 2 — Documentation structure

**Purpose:** One navigable doc tree for maintainers and demo viewers — **after** product truth is frozen.

### Tasks

| ID | Task | Exit |
|----|------|------|
| D2-1 | **Canonical index** — single entry from `docs/README.md` → program pipeline + scope | No competing “start here” paths |
| D2-2 | **Retire redundancy** — merge or archive superseded scope/handoff duplicates; mark stale queues ([v1.5.0-known-issues-and-investigation-queue.md](./v1.5.0-known-issues-and-investigation-queue.md)) as historical | `pnpm docs:check` green |
| D2-3 | **Limitations sheet** — ACC-01/02 + Phase 1 accepts in one presenter-facing doc linked from gate prep | Ready for Phase 5 embed |
| D2-4 | **SQLite / platform honesty** — P3 gap owners documented per [obscur-native-sqlite-policy.md](./obscur-native-sqlite-policy.md) | Gap list closed or accepted |
| D2-5 | **Install/build docs** — [local-desktop-packaging.md](./local-desktop-packaging.md), Android runbook, signing policy aligned with Phase 3 decision | No contradictory install stories |

**Phase 2 exit:** Maintainer can answer “what works, what doesn’t, how to build” from **≤3 hops** from `docs/README.md`. No Phase 3–5 work required to understand product limits.

## Phase 3 — Production installers

**Purpose:** Binaries a demo host can hand to a technical audience.

| ID | Task | Exit |
|----|------|------|
| P3-1 | **Desktop** — `pnpm desktop:package` on Phase 1 commit; artifact under `release-assets/` | NSIS (or documented platform) built; checksum recorded |
| P3-2 | **Signing** — minisign + updater keys **or** explicit unsigned policy in docs | [local-signing-strategy.md](./local-signing-strategy.md) SIGN rows **done** or **accepted unsigned** with honest copy |
| P3-3 | **Android** — release or documented debug demo APK per Phase 1 register outcome | APK path + checksum; Tier 1 re-run on packaged build if needed |
| P3-4 | **Update channel** — repo/release-assets policy documented | Matches [maintainer-distribution-policy.md](./maintainer-distribution-policy.md) |

**Phase 3 exit:** Installers reproducible from documented commands; checksums and signing fingerprints (if any) recorded for website Phase 4.
