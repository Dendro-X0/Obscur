## Why the prior plan failed

| Pattern | Effect |
|---------|--------|
| **Implement ✓ / Verify ◐** while shell crashes | Illusion of 95% done |
| **Policy “SQLite only”** with **chat-state still in 60+ files** | Dual truth; DM loss; unfixable by local patches |
| **Incremental relay/runtime patches** | Unit tests pass; Tauri client still hits `RootErrorBoundary` |
| **Six-phase pipeline starting with features** | Packaging/docs/demo before a stable runtime |
| **Conflicting architecture docs** ([obscur-product-shell-architecture-2026-05.md](./obscur-product-shell-architecture-2026-05.md) “no rewrite” vs [radical-overhaul-v2-target.md](../architecture/radical-overhaul-v2-target.md) “explicit everything”) | No consensus; agents default to passive tuning |

**Feasibility rule:** If a rewrite band shows no user-visible stability after **three substantial iterations**, stop patch loops per [rules/11-feasibility-and-modular-safety.md](../../rules/11-feasibility-and-modular-safety.md) and re-charter or archive.

## Phase 1 — R-SHELL (runtime / shell rewrite)

**Purpose:** One stable React runtime — no fatal loops on nav.

**Strategy:** **Delete feedback paths**, not add guards. Reference: [radical-stability-slice-2026-05.md](./radical-stability-slice-2026-05.md), [ui-effect-stability-policy.md](./ui-effect-stability-policy.md).

| ID | Rewrite | Delete / forbid |
|----|---------|-----------------|
| S1 | **Single relay primary failover owner** | Hook auto-reconcile; failover→recovery loops |
| S2 | **RelayProvider ↛ full window runtime** | `useWindowRuntime` in relay bootstrap |
| S3 | **Route-domain providers** per [obscur-product-shell-architecture-2026-05.md](./obscur-product-shell-architecture-2026-05.md) shell contract | Messaging stack on `/settings` mount path |
| S4 | **Split or gut `NostrMessengerContent`** (`main-shell.tsx`) | 4000-line god component with off-route hooks |
| S5 | **Automated stability gate** | `pnpm verify:stability` + `main-shell` route churn + **Tauri headless/smoke** (add if missing) |

**Exit (all required):**

- [ ] `pnpm verify:stability` green
- [ ] `pnpm -C apps/pwa exec vitest run app/features/main-shell/main-shell.test.tsx -t UV-RUNTIME-1` green
- [ ] Tauri dev: scripted smoke (unlock → settings → network → `/`) **zero** max-depth / hooks errors — **automated**, not maintainer manual
- [ ] [unified-verification-issues-register.md](./unified-verification-issues-register.md): R-SHELL = Pass with SHA

**Forbidden:** New providers, “recover shell” UX as substitute for prevention.

## Phase 2 — R-SQLITE (native persistence rewrite)

**Purpose:** SQLite is the **only** durable read/write authority on native — not policy text.

**Canonical policy:** [obscur-native-sqlite-policy.md](./obscur-native-sqlite-policy.md)

| ID | Rewrite | Delete (native), not bridge |
|----|---------|----------------------------|
| Q1 | **DM write:** every outgoing/incoming path → `db_insert_message` before UI success | Fire-and-forget chat-state writes as truth |
| Q2 | **DM read:** hydrate window from SQLite only | `dm-conversation-native-outgoing-repair`, projection-as-authority |
| Q3 | **Conversation list:** `db_get_conversations` only | `createdConnections` / chat-state list authority |
| Q4 | **Deletes:** tombstone + row delete awaited | Parallel IDB purge |
| Q5 | **Groups cache:** native group tables only | IDB group projections on native |
| Q6 | **Lint/CI:** fail on new native chat-state reads | `NON_CANONICAL` diagnostic → CI |

**Exit:**

- [ ] Grep audit: no `chatStateStoreService` read on native paths in DM hydrate/list (allow explicit migration shims **behind** `hasNativeRuntime() === false` only)
- [ ] Automated: send 10 msgs each direction → restart app → count unchanged (desktop test)
- [ ] Register: R-SQLITE = Pass with SHA

**Forbidden:** “Repair on hydrate” as permanent architecture.
