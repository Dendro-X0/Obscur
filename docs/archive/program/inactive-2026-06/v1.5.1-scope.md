## Goal

Ship **v1.5.1** as a trust patch on the **Nostr client** application layer:

1. **Hide on this device** stays hidden across refresh and account restore, with **show again**.
2. **Honest copy** — no false “erase on the network” claims.
3. **No “Recall for everyone” UI** — not ecosystem-guaranteed on Nostr; ingress may still honor peer delete commands ([dm-recall-for-everyone.md](../messaging/dm-recall-for-everyone.md)).
4. **Same-process profile isolation** — profile-scoped message bus and account-sync mutations (no cross-window UI leaks).

**Bar for success:** users trust local hide; maintainers can describe Nostr limits candidly; the client is **getting better**, not “finished sovereignty.”

## Historical reference — v1.3.15 invariants (logic only)

Release **v1.3.15** documented a simpler delete-for-everyone era. **Do not copy files** — use `git show v1.3.15:<path>` or [GitHub tag v1.3.15](https://github.com/Dendro-X0/Obscur/releases/tag/v1.3.15) for read-only history. Preserve these behaviors on the current stack:

| Invariant | v1.3.15 behavior (reference) | v1.5.1 mapping |
|-----------|-------------------------------|----------------|
| **I1 — Wide target match** | Delete command matches `id`, `eventId`, and `e` tags; searches memory + `messageQueue` | `expandDmDeleteIdsForThread` + identity set at gateway boundary |
| **I2 — Suppress before paint** | Tombstone / filter before UI list shows row again | Write suppression before hydrate returns messages; bus overlay reads suppression |
| **I3 — Sender self-echo** | Self-authored delete command applies locally in ingress | Publish path must call same `applyDmThreadRedaction` / gateway suppression as receive |
| **I4 — Honest scope** | Felt “gone here”; not true global erase | UI: “Hide on this device”; no “Delete for everyone” until v1.6 |

Key reference paths at tag `v1.3.15` (read-only via git):

- `apps/pwa/.../incoming-dm-event-handler.ts` — `applyDeleteCommand`
- `apps/pwa/.../use-chat-actions.ts` — `deleteMessageForEveryone` / `deleteMessageForMe`
- `apps/pwa/.../message-delete-tombstone-store.ts`

Current experimental cooperative path: `apply-dm-thread-redaction.ts`, `dm-redaction-display-gate.ts`, `messaging-client-operations` hydrate pipeline.

## Architecture constraints (AGENTS.md)

1. **One owner per lifecycle** — hide writes only through gateway materialization boundary.
2. **Evidence-based sync** — do not mark hide “synced to all devices” without checkpoint evidence.
3. **No second hydrate owner** — extend `use-conversation-messages` + pipeline, do not add controller-memory filter.
4. **Subtract parallel paths** — if hide touches chat-state, indexed, and projection separately from UI, consolidate.
