# v1.3.8 Offline UI Asset Inventory and Local-First Policy

Last Updated (UTC): 2026-04-05  
Status: Active  
Scope: Phase M1 baseline for hybrid offline-first UI shell

## Purpose

This file is the canonical inventory and policy contract for UI asset dependencies in the v1.3.8 lane.

1. classify shell-critical assets as `local/offline-required` or `network-optional`,
2. define the owner boundary where local-first guarantees are enforced,
3. provide a CI-checkable guard to prevent accidental remote shell regressions.

## Classification Contract

1. `local/offline-required`: must resolve without network for shell boot/navigation surfaces.
2. `network-optional`: may use network for enrichment, but must degrade visibly without blocking shell usability.

## Canonical Owner Boundary

The local-first shell owner remains `apps/pwa` runtime shell + service worker/cache boundary:

1. `apps/pwa/app/layout.tsx`
2. `apps/pwa/app/components/pwa-service-worker-registrar.tsx`
3. `apps/pwa/app/globals.css`
4. `apps/pwa/public/manifest.webmanifest`
5. `apps/pwa/next.config.ts` (`next-pwa` owner wiring)

No feature-level module should introduce a parallel shell asset owner.

## Inventory

| Surface | Owner Files | Runtime Dependency | Classification | Contract |
| --- | --- | --- | --- | --- |
| Root HTML shell, metadata icons, theme/bootstrap scripts | `apps/pwa/app/layout.tsx` | Local manifest and local icon routes/static files (`/manifest.webmanifest`, `/icon-*.png`, `/apple-touch-icon.png`, `/obscur-logo-*.svg`) | `local/offline-required` | Must not embed remote asset URLs in shell metadata. |
| Font runtime delivery | `apps/pwa/app/globals.css` (`--font-geist-sans`, `--font-geist-mono`) | Local CSS font stack only | `local/offline-required` (runtime) | Runtime shell must not depend on live font CDNs or `next/font/google` owner imports. |
| Global UI styling | `apps/pwa/app/globals.css` | Local CSS only | `local/offline-required` | No remote CSS `@import url(http...)`. |
| PWA manifest and install icons | `apps/pwa/public/manifest.webmanifest` + icon route/static files | Root-local icon paths | `local/offline-required` | `start_url` and `scope` stay `/`; icon sources remain rooted local paths. |
| Service worker registration | `apps/pwa/app/components/pwa-service-worker-registrar.tsx` | Local `/sw.js` registration (non-native runtime) | `local/offline-required` | Must keep deterministic local registration path and native-runtime guard. |
| Shell caching strategy | `apps/pwa/next.config.ts` | `next-pwa` generated worker (`public/sw.js`) | `local/offline-required` | Keep caching owner in shell boundary; no feature-owned cache registration. |
| Download-center release metadata | `apps/pwa/app/download/page.tsx` | `https://api.github.com/.../releases/latest` | `network-optional` | Must not block core shell routing/startup; degrade to loading/fallback UI. |
| Link preview enrichment | `apps/pwa/app/api/link-preview/route.ts` | Remote target fetch for URL metadata | `network-optional` | Failure returns fallback JSON; no shell-owner coupling. |
| Profile search fallback API | `apps/pwa/app/features/search/services/profile-search-service.ts` | `https://api.nostr.band/...` fallback | `network-optional` | Relay/local search remains primary path; API failure is non-fatal. |
| Remote media/avatar rendering | `apps/pwa/app/components/optimized-image.tsx` | User/content-provided URLs | `network-optional` | UI must preserve usable fallback when media fetch fails. |

## CI Guard

Local-first shell policy is enforced by:

1. `scripts/check-offline-ui-asset-policy.mjs`,
2. `pnpm offline:asset-policy:check`,
3. CI integration in:
   - `scripts/pwa-ci-scan.mjs`,
   - `scripts/run-release-test-pack.mjs`.

Guard checks:

1. no remote URL literals in shell-critical files,
2. no remote CSS imports in global shell styles,
3. manifest `start_url`/`scope` contract and local icon-only sources,
4. local existence of manifest/layout icon asset references,
5. service worker registration path contract (`/sw.js`) with native-runtime gate.
