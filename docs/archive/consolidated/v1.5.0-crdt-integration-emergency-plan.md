# v1.5.0 CRDT Integration Emergency Plan

**Status:** CRITICAL - Runtime failures block v1.5.0 release  
**Last Updated (UTC):** 2026-05-04 16:30  
**Trigger:** Manual two-user A/B verification discovered critical blockers that v1.4.0 CRDT rewrite was supposed to fix

---

## Investigation Findings

### Current State: Dual Architecture Problem

The project has **two parallel architectures** running simultaneously:

**Architecture A (Old Centralized - BROKEN):**
- Snapshot → Projection → UI pattern
- Event replay as commands
- Single truth arbitration ("which source wins?")
- Polling-based state updates
- File: `apps/pwa/app/features/groups/providers/group-provider.tsx` (still using old patterns)

**Architecture B (New CRDT - PARTIALLY INTEGRATED):**
- CRDT containers → Derived View → UI pattern
- CRDT merge semantics (OR-Set, LWW-Register, G-Counter)
- Union all sources (no arbitration)
- Gossip-based real-time updates
- Files: `apps/pwa/app/features/groups/services/community-membership-crdt.ts` (exists but not fully wired)

**Feature Flags:**
```typescript
export const FEATURE_FLAGS = {
  useCRDTMembership: true,      // CRDT enabled
  logCRDTOperations: true,     // Logging enabled
  enableGossipSync: false,     // Gossip DISABLED
} as const;
```

### The Problem: Incomplete Integration

The v1.4.0 CRDT Protocol Rewrite Roadmap claims:
- Status: "All Phases Complete — RELEASED as v1.4.0"
- All tests passing (153/153 total across all phases)
- All gates met

**BUT runtime verification reveals the exact same blockers:**
1. Phantom calls (historical voice records treated as new)
2. User B cannot see User A messages (B→A visibility gap)
3. Group invitations invisible (relay issues)
4. Historical messages triggered as new notifications

**Root Cause Analysis:**
The CRDT architecture was **implemented but not integrated**. The code exists, but:
- Old centralized code paths remain active
- Feature flags maintain dual compatibility
- Compatibility bridges prevent clean break
- Gossip sync is disabled (`enableGossipSync: false`)
- Runtime behavior still uses old patterns in critical paths

### Why Previous Fixes Failed

The ghost call fix (5-minute age check) is insufficient because:
- It's a patch on the old event-replay architecture
- The fix is only called in one code path (main-shell.tsx line 2302)
- Historical invites from other code paths bypass the check
- The fundamental issue is event replay as commands, not age validation

The B→A visibility gap fix is insufficient because:
- It's a patch on the snapshot arbitration logic
- The fix tries to improve "which source wins" arbitration
- The fundamental issue is that arbitration is the wrong paradigm
- CRDT union semantics are needed, not better arbitration

## Radical Integration Plan
