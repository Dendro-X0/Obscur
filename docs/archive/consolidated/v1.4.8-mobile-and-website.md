# v1.4.8 Roadmap: Mobile Installers & Official Website

## Overview

Release v1.4.8 focuses on expanding distribution channels and establishing a professional web presence.

**Goals**:
1. Enable mobile app distribution through automated CI builds
2. Build a complete official website for product marketing and downloads

---

## Goal 1: Mobile CI Integration (P0)

### Background

Android and iOS build workflows exist but are only triggered manually (`workflow_dispatch`). For v1.4.8, these need to be fully integrated into the release pipeline.

### Deliverables

#### 1.1 Android Automated Builds

**Required**: Build and sign Android APK/AAB as part of release workflow.

**Changes**:
- [ ] Integrate `build-android.yml` into `release.yml` workflow
- [ ] Build universal APK for direct distribution
- [ ] Build AAB for Play Store submission
- [ ] Sign artifacts with release keystore
- [ ] Upload to GitHub Release assets

**Artifacts**:
- `Obscur_1.4.8.apk` - Universal APK for sideloading
- `Obscur_1.4.8.aab` - Play Store bundle

**Canonical Owner**: `.github/workflows/release.yml`

#### 1.2 iOS Automated Builds

**Required**: Build iOS IPA as part of release workflow (requires macOS runner).

**Changes**:
- [ ] Integrate `build-ios.yml` into `release.yml` workflow
- [ ] Build unsigned IPA for TestFlight/App Store
- [ ] Handle provisioning profile (optional signing)
- [ ] Upload to GitHub Release assets

**Artifacts**:
- `Obscur_1.4.8.ipa` - iOS application archive

**Canonical Owner**: `.github/workflows/release.yml`

#### 1.3 Mobile Signing Infrastructure

**Required**: Ensure signing secrets are configured.

**Secrets Needed**:
- `ANDROID_KEYSTORE_BASE64` - Base64-encoded keystore
- `ANDROID_KEYSTORE_PASSWORD` - Keystore password
- `ANDROID_KEYALIAS` - Key alias
- `ANDROID_KEY_PASSWORD` - Key password
- `APPLE_DEVELOPMENT_TEAM` - Apple Team ID (for iOS)
- `APPLE_PROVISIONING_PROFILE` - Provisioning profile (optional)

## Goal 2: Official Website (P0)

### Background

The current website (`apps/website`) is minimal - just a basic landing page and download link. v1.4.8 needs a complete professional website.

### Deliverables

#### 2.1 Website Architecture

**Structure**:
```
/
├── /              - Landing page (features, hero, CTA)
├── /download      - Download page (all platforms)
├── /changelog     - Version history and updates
├── /features      - Feature showcase pages
│   ├── /privacy  - Privacy-first architecture
│   ├── /relay    - Relay-based messaging
│   ├── /voice    - Voice calls
│   └── /groups   - Communities/groups
├── /docs          - User documentation
└── /faq           - Frequently asked questions
```

#### 2.2 Landing Page

**Required**: Compelling hero section with clear value proposition.

**Sections**:
- Hero: "Privacy-first messaging for everyone"
- Feature highlights (3-4 key features with icons)
- Platform availability badges
- Download CTA
- Security/privacy trust badges

**Canonical Owner**: `apps/website/src/app/page.tsx`

#### 2.3 Download Page

**Required**: Central download hub with all platform options.

**Changes**:
- [ ] Platform cards (Desktop, Mobile, Web)
- [ ] Auto-detect user platform
- [ ] Version selector (stable/beta)
- [ ] SHA256 checksums for verification
- [ ] QR code for mobile downloads

**Platforms**:
- **Desktop**: Windows (.exe), macOS (.dmg), Linux (.AppImage)
- **Mobile**: Android (.apk), iOS (TestFlight/App Store)
- **Web**: PWA link, browser requirements

**Canonical Owner**: `apps/website/src/app/download/page.tsx`

#### 2.4 Changelog Page

**Required**: Version history with highlights.

**Changes**:
- [ ] Parse CHANGELOG.md for version entries
- [ ] Show v1.4.7 highlights prominently
- [ ] Collapsible previous versions
- [ ] RSS feed for updates

**Canonical Owner**: `apps/website/src/app/changelog/page.tsx`

#### 2.5 Feature Showcase Pages

**Required**: Deep dives into key product features.

**Pages**:
- `/features/privacy` - E2EE, no servers, self-custody
- `/features/relay` - Nostr relay network, censorship resistance
- `/features/voice` - Encrypted voice calls
- `/features/groups` - Community features

**Canonical Owner**: `apps/website/src/app/features/[slug]/page.tsx`

#### 2.6 Design System

**Required**: Consistent branding and styling.

**Components**:
- Navigation header with download CTA
- Footer with links and social
- Feature card component
- Platform badge component
- Version badge component

**Tech Stack**:
- Next.js 15 App Router
- Tailwind CSS
- shadcn/ui components
- Lucide icons
- Static export for GitHub Pages/Vercel

## Milestone Slices

### M1: Mobile CI (Week 1)

**Goal**: Working mobile builds in release workflow.

**Deliverables**:
- [ ] Android builds integrated and tested
- [ ] iOS builds integrated and tested
- [ ] Signed artifacts uploaded to releases

**Gates**:
- Android APK installs successfully on device
- iOS IPA can be sideloaded (or TestFlight ready)

### M2: Website Structure (Week 2)

**Goal**: Complete website with all pages.

**Deliverables**:
- [ ] Landing page redesigned
- [ ] Download page with all platforms
- [ ] Changelog page functional
- [ ] Feature showcase pages

**Gates**:
- All pages render correctly
- Mobile responsive
- Dark mode support
- Fast load times (<2s)

### M3: Integration & Release (Week 3)

**Goal**: Website deployment integrated with release.

**Deliverables**:
- [ ] Website auto-deploys on release
- [ ] Download links point to latest release assets
- [ ] Version numbers sync automatically

**Gates**:
- Website deploys on v1.4.8 tag
- Download buttons fetch correct assets
- Mobile installers available
