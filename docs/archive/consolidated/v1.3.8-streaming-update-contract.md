# v1.3.8 Streaming Update Contract

Last Updated (UTC): 2026-04-05

## Purpose

Define one canonical contract for desktop in-app updates so rollout controls, safety checks, and fallback behavior are deterministic across UI and release gates.

## Canonical Inputs

1. Signed updater endpoint (`latest.json`) from `apps/desktop/src-tauri/tauri.conf.json`.
2. Streaming policy manifest contract (`apps/desktop/release/streaming-update-policy.example.json`).
3. Desktop bundle config must publish updater artifacts (`bundle.createUpdaterArtifacts = true`) so release workflow can generate a valid updater feed.

## Manifest Contract Fields

Required fields:

1. `version` semver-like string.
2. `channel` in `stable|beta|canary`.
3. `rolloutPercentage` in `0..100`.
4. `killSwitch` boolean.
5. `artifacts` map keyed by platform with:
   - `url` (https),
   - `signature` (non-empty),
   - `checksumSha256` (64 hex chars).

Optional fields:

1. `minSafeVersion` semver-like string.
2. `releaseNotesUrl` (http/https).

## Runtime Policy Decisions

`apps/pwa/app/features/updates/services/streaming-update-policy.ts` resolves:

1. channel eligibility,
2. kill-switch block,
3. staged rollout holdback (deterministic per-device seed),
4. minimum-safe-version escalation (`forceUpdateRequired`),
5. rollback behavior contract (`preserve_current_version`).

## Failure Contract

Install failure classification preserves current version in all failure classes:

1. `verification_failed` (signature/checksum/integrity),
2. `download_failed`,
3. `install_failed`,
4. `no_update_available`,
5. `unknown_failure`.

## Gate Integration

1. `pnpm release:streaming-update-contract:check` validates updater config + manifest schema.
2. Release workflow now generates two streaming-update assets from downloaded desktop artifacts:
   - `streaming-update-policy.json`
   - `latest.json`
3. `scripts/run-release-test-pack.mjs` and `scripts/release-preflight.mjs` execute the check as a release gate.
4. Focused tests:
   - `apps/pwa/app/features/updates/services/streaming-update-policy.test.ts`
   - `apps/pwa/app/features/updates/services/release-download-targets.test.ts`
   - `apps/pwa/app/components/pwa-service-worker-registrar.test.tsx`
