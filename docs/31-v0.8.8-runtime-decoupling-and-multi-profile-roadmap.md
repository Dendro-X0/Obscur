# v0.8.8 Runtime Decoupling + Multi-Profile Foundation

_Last reviewed: 2026-03-06 (baseline commit 51a511b)._

## Summary

v0.8.8 focuses on two stabilization tracks:

1. Runtime decoupling between native and web codepaths.
2. Single-active multi-profile foundation on one device.

Locked policy in this iteration:

- Web runtime policy: dev/localhost allowed, production web blocked by default.
- Profile model: single active profile.
- Migration: one-time backup + auto-migrate legacy single-profile state to `default`.

## Delivered Scope

### V088-ARCH-001 Capability Contract

- Added shared runtime capability contract:
  - `RuntimeCapabilities`
  - `getRuntimeCapabilities()`
  - `hasNativeRuntime()`
- Tightened native detection to require callable bridge signals (not marker objects alone).

### V088-ARCH-002 Adapter Layer Split

- Added shared adapter result contract:
  - `AdapterResult<T>`
  - `okResult`, `unsupportedResult`, `failedResult`
- Added native command adapter:
  - `invokeNativeCommand(...)`
- Updated key native-sensitive paths to use capability checks and deterministic fallback behavior.

### V088-ARCH-003 Web Runtime Policy

- Added `NativeRuntimeGate` policy behavior:
  - Dev + localhost web allowed.
  - Production web blocked by default.
  - `NEXT_PUBLIC_ENABLE_WEB_RUNTIME_PROD=1` can explicitly allow production web runtime.
- Added explicit blocked-runtime screen messaging.

## Profile Isolation Foundation

### V088-PROFILE-001 Profile Registry

- Added profile registry service:
  - `ProfileRecord`
  - `ActiveProfileState`
  - `ProfileSwitchResult`
- Supports create/rename/switch/remove with active-profile invariant enforcement.

### V088-PROFILE-002 Namespaced Persistence

- Added profile scoping helpers for storage and identity keys.
- Scoped persistence now applied to core local state:
  - identity IndexedDB key
  - privacy settings
  - profile draft/settings storage
  - NIP-96 provider config key
  - local media storage config/index
- Extended scoped+legacy-fallback coverage to additional state:
  - relay persistence cache
  - offline manager state
  - dev-mode toggle key
  - invite cache and invite secure-storage helpers
  - group tombstones and group notification/muted-member preferences
  - PIN lock storage and remember-me auth token flow

### V088-PROFILE-003 Identity + Key Isolation

- Identity storage key is now profile-scoped (`identity::<profileId>`).
- Identity hook listens for profile-change events and rehydrates as locked state for target profile.

### V088-PROFILE-004 Switcher UX

- Added Settings identity profile switcher UI:
  - create profile
  - rename active profile
  - switch profile
  - remove profile (non-default)
- Switch flow enforces lock-before-switch and runtime reload.

## Migration + Hardening

### V088-MIG-001 One-Time Migration

- Added one-time migration service and startup bootstrap:
  - snapshot backup in localStorage
  - migrate legacy keys into `default` profile namespace
  - migrate legacy identity key (`primary`) to `identity::default`
  - idempotent marker + migration report

### V088-REL-001 Runtime Error Hygiene

- Reduced web console error spam by preventing unsupported Tauri command invocation in web runtime.
- Updated desktop shell rendering checks to avoid false-positive native UI rendering in web.

## Validation

Targeted tests added for:

- runtime capability derivation
- profile registry invariants and switching behavior
- migration backup/idempotency behavior
- scoped storage compatibility:
  - `pin-lock-service` scoped + legacy fallback
  - `group-tombstone-store` scoped + legacy fallback

## Operational Notes

- Profile remove is local-only and best-effort for profile-scoped local state.
- Web remains a development/test harness in v0.8.8, not a default production target.
