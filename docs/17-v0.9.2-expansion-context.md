## Emergency Release Constraint (2026-03-20)

- v0.9.2 is shipping with known critical degradations to avoid further destabilizing last-minute rewrites.
- The project is currently in a recurrence pattern:
: - new fixes can surface new failures,
: - previously mitigated failures can reappear after adjacent lifecycle/sync changes.
- Current high-risk runtime failures to carry into next iteration:
: 1. login-state persistence regression (remember-me/session restore unreliable),
: 2. page-transition freeze and sidebar interaction lock,
: 3. infinite loading loops after startup/profile disruption,
: 4. recurring cross-device self-authored DM history loss,
: 5. historical media hydration mismatch across runtimes.

Release policy implication:
- do not represent v0.9.2 as stable for full historical integrity and navigation liveness.

## What Is Stable Enough To Build On

1. Cross-device account backup ordering and restore selection now use deterministic ordering contracts.
2. Group/community membership recovery uses explicit precedence and reconstruction boundaries.
3. DM restore paths preserve self-authored history in canonical merge/apply + append flows.
4. Startup now has fail-open guards:
: - identity IndexedDB open timeout/blocked handling,
: - profile boot stall recovery path with explicit user action.
5. Release reliability gate is currently green for this snapshot:
: - `pnpm release:test-pack -- --skip-preflight` passes.

## What Is Still Fragile

1. UI responsiveness during relay/runtime turbulence:
: route transitions can still freeze under sustained churn.
2. Startup/profile mismatch edge cases:
: accidental key/profile switching can still produce degraded startup states that require recovery actions.
3. Observability-to-action gap:
: logs are available, but diagnosis still depends on disciplined boundary capture.
4. Login persistence continuity:
: auto-login and remembered-session continuity remains unstable after lifecycle changes.
5. Historical media replay:
: desktop and web can diverge on historical attachment hydration after account restore.
