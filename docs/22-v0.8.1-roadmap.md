# v0.8.1 Ticketized Roadmap

_Last reviewed: 2026-03-04 (baseline commit b8cc5fe)._

_Drafted: 2026-03-04._

This roadmap defines a stabilization-first `v0.8.1` release.  
Primary direction: improve UX/reliability/performance of existing features, with only low-risk functional expansion.

## 1) Release Theme

1. Stabilize and polish current capabilities before major expansion.
2. Reduce startup/runtime noise and improve user trust in error feedback.
3. Improve chat/media smoothness under realistic load.
4. Keep release operations deterministic.

## 2) Scope and Guardrails

### In scope

1. UX clarity improvements for current flows (chat, media, relays, startup).
2. Reliability hardening in DM/group processing and media playback.
3. Performance tuning/verification on low-end profile.
4. CI/release determinism for GitHub Release artifacts.

### Out of scope

1. New protocol layers or major schema migrations.
2. Large new feature surfaces that increase release risk.
3. App-store deployment work requiring signing infra setup.

## 3) Ticket Backlog

## A. UX + Runtime Signal Quality

1. `V081-UX-001` Startup Log Hygiene Finalization (P0, S)
   - Goal: remove repeated expected-noise logs in dev and production startup paths.
   - Deliverables:
     - rate-limit or downgrade known non-actionable warnings.
     - keep true failures surfaced with actionable context.
   - Acceptance:
     - clean startup on desktop dev: no repeated red error overlays from expected conditions.
     - warning/error stream reflects actionable issues only.

2. `V081-UX-002` Media Error Copy and Recovery UX (P0, M)
   - Goal: unify media error behavior and recovery actions.
   - Deliverables:
     - consistent fallback card/copy for audio/video/image load failures.
     - explicit retry/open/download affordances.
   - Acceptance:
     - media failures no longer appear as generic console panic.
     - manual retry and open-external always available.

3. `V081-UX-003` Relay Degraded-State UX Pass (P0, M)
   - Goal: distinguish degraded relay scope from hard failure.
   - Deliverables:
     - concise status labels and recommended action links in settings/chat surfaces.
   - Acceptance:
     - users can tell whether messaging is partially degraded vs fully unavailable.

4. `V081-UX-004` Theme/Contrast Polish for Updated Components (P1, S)
   - Goal: verify light/dark readability for updated media/chat cards and status badges.
   - Deliverables:
     - contrast fixes where needed.
   - Acceptance:
     - no low-contrast text regressions in updated messaging UI.

## B. Messaging Reliability

1. `V081-REL-001` Decrypt-Failure Classification Hardening (P0, M)
   - Goal: classify expected foreign/malformed events separately from true crypto regressions.
   - Deliverables:
     - categorized error buckets and bounded telemetry.
   - Acceptance:
     - no user-facing failure path for expected foreign decrypt misses.
     - actionable alerts remain for real regressions.

2. `V081-REL-002` Subscription Lifecycle Idempotency Pass (P0, M)
   - Goal: prevent duplicate subscribe/close churn on startup/network transitions.
   - Deliverables:
     - stable subscription guards and lifecycle logging hygiene.
   - Acceptance:
     - no repeated subscribe/close thrash for same session lifecycle.

3. `V081-REL-003` Attachment Cache Path Resilience (P0, M)
   - Goal: avoid noisy failures when local cache path permissions are unavailable.
   - Deliverables:
     - explicit fallback path with non-blocking UX.
   - Acceptance:
     - message/media flow continues when local cache write fails.
     - warning is single, rate-limited, and actionable.

## C. Performance and Responsiveness

1. `V081-PERF-001` Scroll Smoothness Acceptance Sweep (P0, M)
   - Goal: validate current performance mode in real conversations.
   - Deliverables:
     - measured runs for DM and group timelines with media-heavy windows.
   - Acceptance:
     - median FPS and p95 update latency meet current phase targets.

2. `V081-PERF-002` Synthetic Load Tooling Hardening (P0, M)
   - Goal: make seeded-history and burst testing repeatable by maintainers.
   - Deliverables:
     - standardized dev commands for 10k seed + burst scenarios.
   - Acceptance:
     - one-command local load setup documented and reproducible.

3. `V081-PERF-003` Message List Rendering Micro-optimizations (P1, M)
   - Goal: reduce avoidable render work in heavy timelines.
   - Deliverables:
     - targeted memoization/derived-data caching improvements.
   - Acceptance:
     - reduced commit cost under rapid scroll + incoming bursts.

## D. Release and Operations Discipline

1. `V081-OPS-001` Release Workflow Determinism (P0, S)
   - Goal: ensure one authoritative release workflow for tag-based artifacts.
   - Deliverables:
     - no duplicate tag release publishers.
     - stable desktop + keyless mobile artifact handling.
   - Acceptance:
     - tag run produces expected artifacts without workflow-file failures.

2. `V081-OPS-002` Artifact Matrix Verification (P0, S)
   - Goal: verify exact artifact names/paths for `v0.8.1`.
   - Deliverables:
     - release checklist updated with expected files per platform.
   - Acceptance:
     - GitHub release contains all expected desktop installers and available mobile outputs.

3. `V081-OPS-003` Release Runbook Snapshot and Evidence (P1, S)
   - Goal: capture commands/results/risks for `v0.8.1` go-no-go.
   - Deliverables:
     - evidence log and rollback notes.
   - Acceptance:
     - reproducible release package steps documented.

## E. Low-Risk Functional Expansion (Allowed but Limited)

1. `V081-EXP-001` Media Compatibility Increment Pack (P1, M)
   - Goal: add missing MIME handling and fallback behavior for already-supported document/media categories.
   - Deliverables:
     - compatibility matrix updates without new protocol scope.
   - Acceptance:
     - no regressions in existing media/document send-preview-render pipeline.

## 4) Suggested Sequencing

1. Wave 1 (Days 1-2): `UX-001`, `REL-001`, `REL-002`, `OPS-001`.
2. Wave 2 (Days 3-5): `UX-002`, `UX-003`, `REL-003`, `PERF-001`.
3. Wave 3 (Days 6-7): `PERF-002`, `PERF-003`, `OPS-002`.
4. Wave 4 (Final): `UX-004`, `OPS-003`, optional `EXP-001`, freeze.

## 5) Exit Criteria for v0.8.1

1. All P0 tickets complete.
2. No startup red-overlay errors from expected runtime conditions.
3. No known message loss/order regressions in DM/group burst tests.
4. Release workflow for tag builds is green and artifact-complete.
5. Docs/changelog/runbook updated with shipped behavior.

## 6) Evidence Required per Ticket

1. Changed file references.
2. Command outputs (tests/checks/profiling).
3. Risk note (remaining failure modes).
4. Rollback note (safe disable/revert path).
