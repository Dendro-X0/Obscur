# 33 v1.1.4 CP4 Voice Long-Session Matrix

_Last reviewed: 2026-03-24 (baseline commit 00683987)._

This matrix defines the `v1.1.4` kickoff slice for `M9` CP4 prep.

Primary objective:
1. provide deterministic long-session replay evidence without adding runtime owners,
2. verify repeated degrade/recovery convergence for realtime voice under sustained cycles,
3. provide an explicit CP4 readiness gate for operator triage.

## Canonical Helper

Preferred one-copy export:
: `copy(window.obscurM6VoiceReplay?.runLongSessionReplayCaptureJson?.({ clearAppEvents: true, captureWindowSize: 400, cycleCount: 6 }))`

Raw object probe:
: `window.obscurM6VoiceReplay?.runLongSessionReplayCapture?.({ clearAppEvents: true, captureWindowSize: 400, cycleCount: 6 })`

Compact CP4 self-test probe (recommended):
: `window.obscurM6VoiceReplay?.runCp4LongSessionSelfTest?.({ clearAppEvents: true, captureWindowSize: 400, cycleCount: 6 })`
: `copy(window.obscurM6VoiceReplay?.runCp4LongSessionSelfTestJson?.({ clearAppEvents: true, captureWindowSize: 400, cycleCount: 6 }))`

Compact CP4 gate probe (recommended for quick pass/fail triage):
: `window.obscurM6VoiceReplay?.runCp4LongSessionGateProbe?.({ clearAppEvents: true, captureWindowSize: 400, cycleCount: 6 })`
: `copy(window.obscurM6VoiceReplay?.runCp4LongSessionGateProbeJson?.({ clearAppEvents: true, captureWindowSize: 400, cycleCount: 6 }))`

Compact CP4 checkpoint bundle (recommended one-copy release-readiness export):
: `window.obscurM6VoiceReplay?.runCp4CheckpointCapture?.({ clearAppEvents: true, captureWindowSize: 400, cycleCount: 6 })`
: `copy(window.obscurM6VoiceReplay?.runCp4CheckpointCaptureJson?.({ clearAppEvents: true, captureWindowSize: 400, cycleCount: 6 }))`

Gate diagnostics event probe:
: `window.obscurAppEvents?.getCrossDeviceSyncDigest?.(400)?.events?.["messaging.realtime_voice.long_session_gate"]`

Gate summary probe:
: `window.obscurAppEvents?.getCrossDeviceSyncDigest?.(400)?.summary?.realtimeVoiceSession`

## Expected Bundle Shape

1. `replay`:
: deterministic long-session replay summary.
2. `capture`:
: M6 one-copy capture bundle.
3. `replayConfig`:
: `cycleCount` and `injectRecoveryExhausted`.
4. `cp4ReadinessGate`:
: aggregate CP4-prep readiness verdict.
5. `cp4CheckpointGate`:
: aggregate checkpoint pass/fail verdict (when using checkpoint helper).

Required fields to verify:
1. `replay.finalState.phase === "active"`
2. `replayConfig.injectRecoveryExhausted === false`
3. `cp4ReadinessGate.pass === true`
4. `cp4ReadinessGate.checks.transitionVolumeSufficient === true`
5. `cp4ReadinessGate.checks.degradedTransitionsSufficient === true`
6. `cp4ReadinessGate.checks.recoveredTransitionsSufficient === true`
7. `cp4ReadinessGate.checks.endedTransitionsZero === true`
8. `cp4ReadinessGate.checks.digestRecoveryExhaustedZero === true`
9. `cp4ReadinessGate.checks.asyncVoiceStartFailureCountZero === true`
10. `cp4ReadinessGate.checks.deleteRemoteFailureCountZero === true`
11. `selfTestGate.pass === true` (when using CP4 self-test helper)
12. `selfTestGate.failedChecks` is an empty array.
13. digest summary reports:
: `longSessionGateCount >= 1`,
: `latestLongSessionGatePass === true`,
: `unexpectedLongSessionGateFailCount === 0`.

## Failure Injection Probe

To verify deterministic failure-path signaling:
: `window.obscurM6VoiceReplay?.runLongSessionReplayCapture?.({ clearAppEvents: true, captureWindowSize: 400, cycleCount: 4, injectRecoveryExhausted: true, maxRecoveryAttempts: 2 })`

Expected:
1. `replay.finalState.phase === "ended"`
2. `replay.finalState.lastTransitionReasonCode === "recovery_exhausted"`
3. `cp4ReadinessGate.pass === false`
4. `cp4ReadinessGate.failedChecks` contains:
: `finalPhaseActive`,
: `endedTransitionsZero`,
: `digestRecoveryExhaustedZero`.

## Pass Criteria

1. Long-session helper returns replay + capture + gate with no missing fields.
2. `cp4ReadinessGate.failedChecks` is empty for nominal replay.
3. Failure injection replay fails deterministically with recovery-exhausted checks.

## Escalation Criteria

Escalate as blocker if either occurs:
1. nominal replay has `cp4ReadinessGate.pass === false`,
2. nominal replay has `replay.finalState.phase !== "active"`,
3. nominal replay shows `recoveryExhaustedCount > 0`,
4. failure injection does not produce `recovery_exhausted` end-state.

Attach in escalation:
1. long-session one-copy capture JSON,
2. `copy(window.obscurM6VoiceCapture?.captureJson(400))`,
3. `copy(window.obscurM0Triage?.captureJson(300))`,
4. short timestamped replay timeline.
