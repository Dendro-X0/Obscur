# v1.5.0 Release Notes

**Codename:** Gateway & Membership  
**Target date:** 2026-05-15  
**Phase:** Release candidate

---

## Summary

v1.5.0 converges client behavior through **ClientGateway**, hardens **profile-scoped runtime**, and delivers **relay-first community membership** with evidence-backed coordinator ownership. It is a stability and architecture release.

**Not in scope for this release:** user-visible “delete for everyone” for DMs (deferred — see [messaging/redaction-v1.5-deferred.md](../messaging/redaction-v1.5-deferred.md)).

---

## Highlights

### ClientGateway (R0–R2)

- DM conversation materialization, suppression prep, and community roster paths route through `getResolvedClientGateway()`.
- Reduced direct `localDmVisibilityOwner` / storage-port reach-through from hooks.
- See [gateway/client-unified-gateway.md](../gateway/client-unified-gateway.md).

### Profile runtime

- `ProfileMessageBus` with profile-scoped event isolation.
- `getResolvedProfileId()` migration across messaging, groups, and shell.
- Same-process A/B isolation tests.

### Community membership (Phase 3)

- `community-membership-ingress` → `applyCommunityMembershipIngress` → coordinator/ledger.
- Provisional join on invite receive; authoritative join on relay gossip evidence.
- Terminal invite decline / inviter observe-only paths.

### Account projection

- DM timeline events (`DM_RECEIVED`, `DM_REMOVED_LOCALLY`) for restore and read cutover.
- Projection-first DM read model when `read_cutover_enabled`.

### Messaging reliability (incremental)

- DM send retry queue and operation ledger (shadow/diagnostics).
- Hybrid relay targeting for delete commands (when re-enabled).
