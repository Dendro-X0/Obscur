# v1.9.11 Release Notes

**Codename:** Vault encryption sandbox (Phases 1–5)  
**Status:** **Release prep** — engineering landed on `main` through `ed17e7bb`  
**Predecessor:** [v1.9.10](./v1.9.10-release.md)

---

## Summary

**v1.9.11** ships the **VAULT-SANDBOX-1** band: vault media is encrypted on disk, legacy plaintext is migrated on unlock, metadata moves into encrypted SQLite, export/decrypt paths are explicit, and new writes use per-profile `profiles/{id}/vault/` layout.

**User-visible:** Secure Upload → encrypted `.obscurvault` blobs on disk; in-app preview while unlocked; explicit export for plaintext. **Chat → Save to Vault** remains hidden (Phase 6 disabled).

---

## What shipped

| Area | Detail |
|------|--------|
| **Phase 1** | Encrypt-on-write hardening — refuse vault writes without PDK |

| **Phase 4** | Export labeling, folder-only reveal, blob URL revoke on lock |
