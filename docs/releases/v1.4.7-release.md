# v1.4.7 Release Notes

**Release Date**: 2026-04-30  
**Codename**: "Resilience & Verification"

## Summary

v1.4.7 delivers the Resilience & Verification milestone, completing critical fixes, M1 implementation goals, and comprehensive diagnostics. This release focuses on:

1. **M0: Critical Fixes** — Ghost call prevention, profile isolation, notification fixes
2. **M1: Implementation Goals** — CAS Media Recovery, Security Integration, Relay Capability Badges
3. **M2: Diagnostics** — Verification utilities and replay test suite
4. **M3: Stabilization** — Type fixes, documentation, release preparation

---

## M0: Critical Fixes (COMPLETE)

### Ghost Call Prevention
- **File**: `realtime-voice-history-replay-policy.ts`
- **Fix**: Added 5-minute TTL to voice call invites to prevent historical invites from reactivating during account sync
- **Impact**: Eliminates ghost calls from restored history

### Profile Isolation Fix
- **File**: `apps/desktop/src-tauri/src/profiles.rs`
- **Fix**: Complete rewrite of profile removal with window closure, file lock release, and shared WebView storage cleanup
- **Impact**: New profiles now start with truly fresh environment, no auto-login with deleted credentials

### Historical Message Notification Fix
- **File**: `desktop-notification-handler.tsx`
- **Fix**: Skip notifications during account sync phase — only notify when `phase === "ready"`
- **Impact**: Historical messages from restore no longer trigger "new message" notifications

## M1: Implementation Goals (COMPLETE)

### Goal 2: CAS Media Recovery (Restore Convergence)
- **Files**: `vault/services/cas-media-recovery.ts`, `account-sync/services/restore-materialization.ts`
- **Feature**: Automatic recovery of missing media blobs after fresh-device restore
- **Process**:
  1. Scan restored messages for CAS attachment references
  2. Check vault for missing blobs
  3. Fetch missing blobs from CAS gateway (`https://cas.obscur.app/blob/{hash}`)
  4. Save to local vault
- **Integration**: Automatically runs after chat state restoration

### Goal 4: Security Integration
- **Files**: `security/services/security-integration.ts`, `messaging/components/message-header.tsx`
- **Features**:
  - Security audit logging (encrypted, append-only)
  - Identicon generation for sender verification
  - Key change detection on incoming messages
  - Security event classification (info/warning/critical)
- **UI**: Message header shows sender identicon and verification status

### Goal 5: Relay Capability Badges
- **Files**: `groups/components/community-mode-badge.tsx`, `groups/components/group-management-dialog.tsx`
- **Features**:
  - Community mode display (Sovereign Room vs Managed Workspace)
  - Relay tier indicators (unconfigured/public_default/trusted_private/managed_intranet)
  - Guarantee descriptions for each mode
  - Caution notices for mode limitations
- **UI**: Badge integrated into group management dialog
