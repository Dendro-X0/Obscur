# v1.4.7 Restore & Media Loss Reproduction Steps

**Status**: Investigation Document  
**Purpose**: Define exact steps to reproduce media loss on fresh-device restore  
**Last Updated**: 2026-04-30

---

## Problem Statement

From open blockers:
> **Fresh-device restore**: media clears from message history / Vault after login

**Expected**: Media attachments in DM history should be preserved through account restore on new device/fresh browser.

**Actual**: Media appears to be lost after restore completes.

---

## Environment Setup

### Test Account (Required)
- Active Obscur account with media in conversations
- At least 3-5 media items (images/videos) across 2+ conversations
- Account backup available on relay/coordination

### Fresh Device/Browser
- Incognito/private window, OR
- Different browser, OR
- Clear site data (IndexedDB, LocalStorage, Cache)

---

## Reproduction Steps

### Phase 1: Establish Baseline on Source Device

1. **Login to existing account**
   ```
   URL: https://obscur.app (or local dev)
   Action: Unlock with existing credentials
   ```

2. **Navigate to conversation with media**
   ```
   Path: Chat → Select conversation with images
   Verify: Thumbnails visible, media loads on click
   ```

3. **Check Vault state**
   ```javascript
   // Run in console
   const vault = await window.obscurVault?.getSnapshot?.();
   console.log("Vault items:", vault?.blobs?.length || 0);
   console.log("Vault metadata:", vault?.metadata?.length || 0);
   ```
   **Record**: Expected: X items in vault

4. **Trigger account backup** (if not recent)
   ```
   Path: Settings → Account → Backup Now
   Wait: "Backup complete" confirmation
   ```

5. **Note conversation media hashes**
   ```javascript
   // Find messages with media
   const messages = await window.obscurMessaging?.getCurrentConversationMessages?.();
   const mediaMessages = messages?.filter(m => m.attachmentHash);
   console.log("Media messages:", mediaMessages?.map(m => ({
     id: m.id,
     hash: m.attachmentHash,
     type: m.attachmentType
   })));
   ```
   **Record**: List of content hashes for verification

---

### Phase 2: Simulate Fresh Device

6. **Clear all stored data**
   ```javascript
   // Nuclear option for testing
   await window.indexedDB.databases().then(dbs => 
     Promise.all(dbs.map(db => window.indexedDB.deleteDatabase(db.name)))
   );
   localStorage.clear();
   sessionStorage.clear();
   ```
   OR manually: DevTools → Application → Clear Storage → Clear site data

7. **Reload app**
   ```
   Action: Hard refresh (Ctrl+Shift+R)
   Verify: Shows onboarding/login screen (not logged in)
   ```

---

### Phase 3: Restore Account

8. **Begin restore flow**
   ```
   Path: Login → "Restore from backup"
   Enter: Recovery phrase / credentials
   ```

9. **Monitor restore progress**
   ```javascript
   // Watch account sync phase
   const checkPhase = () => {
     const phase = window.obscurAccountSync?.getPhase?.();
     console.log("Sync phase:", phase);
     return phase;
   };
   setInterval(checkPhase, 1000);
   ```
   **Record**: Note how long each phase takes

10. **Wait for "Restore complete"**
    ```
    Expected: UI shows "Account restored", chat list appears
    ```

---

### Phase 4: Verify Media State

11. **Check conversations for media**
    ```
    Path: Chat → Same conversation as step 2
    Observe: Thumbnails visible or "missing image" placeholders?
    ```
    **Record**: Screenshot of media state

12. **Check Vault state**
    ```javascript
    const vault = await window.obscurVault?.getSnapshot?.();
    console.log("Vault items post-restore:", vault?.blobs?.length || 0);
    console.log("Vault metadata post-restore:", vault?.metadata?.length || 0);
    ```
    **Record**: Expected: X items (same as step 3) vs Actual: Y items

13. **Check for orphaned media references**
    ```javascript
    // Messages that reference media not in vault
    const messages = await window.obscurMessaging?.getCurrentConversationMessages?.();
    const orphaned = messages?.filter(m => {
      if (!m.attachmentHash) return false;
      const inVault = vault?.blobs?.some(b => b.hash === m.attachmentHash);
      return !inVault;
    });
    console.log("Orphaned media references:", orphaned);
    ```
    **Record**: List of orphaned references

14. **Attempt media fetch**
    ```
    Action: Click on "missing" image thumbnail
    Observe: Does it load from CAS? Timeout? Error?
    ```
    **Record**: Network tab shows CAS fetch attempt

## Expected vs Actual

### Expected Behavior
1. Account metadata restores (profile, contacts, conversations)
2. Vault metadata restores (content hashes, filenames, timestamps)
3. Vault blobs download from CAS (content-addressed storage)
4. Media thumbnails appear in conversations
5. Clicking media displays full image

### Actual Behavior (To Document)
- [ ] **Case A**: Vault empty (0 items) — metadata didn't restore
- [ ] **Case B**: Vault has metadata but no blobs — CAS fetch failed
- [ ] **Case C**: Vault has partial data — race condition or timeout
- [ ] **Case D**: Media appears after delay — lazy loading works
- [ ] **Case E**: Media never appears — complete loss

## Suspected Root Causes (From Code Analysis)

### Hypothesis 1: Vault Activation Timing

**Location**: `apps/pwa/app/features/vault/services/local-media-store.ts`

**Issue**: Vault may become "active" before restore completes, causing premature access.

**Evidence to collect**:
```javascript
// Check vault ready state during restore
const checkVault = () => {
  const vault = window.obscurVault?.getSnapshot?.();
  console.log("Vault ready:", vault?.ready, "Item count:", vault?.blobs?.length);
};
setInterval(checkVault, 500);
```

### Hypothesis 2: CAS Fetch Not Triggered

**Location**: `apps/pwa/app/features/vault/services/cas-media-recovery.ts`

**Issue**: Content hashes restored but CAS fetch not initiated.

**Evidence to collect**:
```javascript
// Check CAS fetch queue
console.log("CAS queue:", window.obscurVault?.casQueue?.length);
console.log("CAS pending:", window.obscurVault?.casPending);
```

### Hypothesis 3: Restore Merge Order

**Location**: `apps/pwa/app/features/account-sync/services/encrypted-account-backup-service.ts`

**Issue**: Conversation messages restore before vault metadata, causing references to missing data.

**Evidence to collect**:
```javascript
// Check restore namespace completion
console.log("Restore namespaces:", window.obscurAccountSync?.getNamespaceStatus?.());
```

### Hypothesis 4: Missing Blob Cleanup

**Note**: Vault cleanup logic is now integrated into `local-media-store.ts`

**Issue**: Aggressive cleanup removes "orphaned" blobs that are actually referenced by not-yet-restored messages.
