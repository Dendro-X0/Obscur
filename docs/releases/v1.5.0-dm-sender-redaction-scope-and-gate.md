# v1.5.0 — DM Sender Redaction Scope and Release Gate

**Status:** Cooperative “delete for everyone” is **not** a v1.5.0 user promise.  
**Product gate:** `DM_DELETE_FOR_EVERYONE_UI_ENABLED = false` in `apps/pwa/app/features/messaging/config/dm-redaction-product.ts`.

---

## Scope (what v1.5.0 includes)

| In scope | Out of scope (v1.5.0) |
|----------|------------------------|
| Delete **for me** (local suppression) | User-visible “delete for everyone” |
| Receive pipeline classifies delete commands (no UI) | Two-profile redaction smoke as release blocker |
| `applyDmThreadRedaction`, display gate (experimental) | Marketing copy implying cooperative erase |
| `pnpm verify:dm-redaction` in CI/maintainer workflow | Community extension / anti-abuse modules |

Narrative: [messaging/redaction-v1.5-deferred.md](../messaging/redaction-v1.5-deferred.md).  
v1.6+ design: [messaging/cooperative-redaction-future.md](../messaging/cooperative-redaction-future.md).

---

## Release gate checklist (v1.5.0 tag)

Engineering must pass **before** tagging; user-facing redaction is **not** required.

| # | Gate | Evidence |
|---|------|----------|
| G1 | UI flag off | `dm-redaction-product.ts` → `DM_DELETE_FOR_EVERYONE_UI_ENABLED === false` |
| G2 | Automated bundle green | `pnpm verify:dm-redaction` |
| G3 | Typecheck | `pnpm -C apps/pwa exec tsc --noEmit` |
| G4 | Docs integrity | `pnpm docs:check` |
| G5 | Release notes honest | [v1.5.0-release.md](./v1.5.0-release.md) + root `CHANGELOG.md` state **deferred** |
| G6 | No false “applied” UX | Menu does not offer “Delete for everyone” |

**Explicitly not gating v1.5.0:** A/B profile smoke where B still sees a redacted message (tracked for v1.6 — [investigation-delete-for-everyone.md](../messaging/investigation-delete-for-everyone.md)).

## v1.6 re-enable criteria (preview)

Do not re-enable UI until:

1. Single canonical message id contract across projection, persistence, and chat read model.
2. One cooperative apply owner (ClientGateway / thread redaction ingress only).
3. Two-profile automated + manual gate passes documented in handoff.
