## Mobile CI Integration

### Android Builds
- **Workflow**: `.github/workflows/release.yml`
- **Outputs**: Unsigned APK and AAB files (signed when secrets available)
- **Assets**: `app-universal-release-unsigned.apk`, `app-universal-release.aab`
- **Signing**: Optional ‚Äî builds always, signs if `ANDROID_KEYSTORE_PASSWORD` secret exists

### iOS Builds
- **Workflow**: `.github/workflows/release.yml`
- **Outputs**: Simulator build (unsigned) or signed IPA when Apple certificates available
- **Assets**: `Obscur_*.ipa` (when signed) or simulator `.app`
- **Signing**: Optional ‚Äî attempts signed build with secrets, falls back to simulator

- Auto-detects platform (mobile vs desktop)
- Shows relevant download section first

## Known Limitations

- **iOS signed builds**: Require Apple Developer certificates (`APPLE_CERTIFICATE_BASE64`, `APPLE_CERTIFICATE_PASSWORD`, `APPLE_DEVELOPMENT_TEAM`)
- **Android signed builds**: Require keystore secret (`ANDROID_KEYSTORE_PASSWORD`)
- Unsigned mobile builds work for testing/sideloading but not App Store distribution
