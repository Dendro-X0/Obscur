## Summary (planned)

v1.5.4 focuses on **desktop/web quality** and **one consistent Obscur logo** on installers. **Mobile device testing is paused** for this cycle; CI may still produce APK artifacts without maintainer M1–M3 sign-off.

---

## Planned highlights

### Unified branding

- Replace Tauri default icons with the Obscur mark used by the PWA (`obscur-logo-dark.svg` / generated `icon-512`).
- Windows NSIS installer, macOS `.dmg`, Linux `.AppImage`, and Android launcher draw from the same regenerated asset set.

### Client polish

- Continue relay connect discipline (WS-C).
- One additional stability or UX fix with regression coverage.

### Install trust (documentation)

Obscur releases from [public GitHub](https://github.com/Dendro-X0/Obscur) are **unsigned** until maintainer configures:

- Windows: Authenticode (`certificateThumbprint` in `tauri.conf.json`)
- Android: `ANDROID_KEYSTORE_*` GitHub secrets for release signing

Until then, Windows SmartScreen and Android “unknown app” warnings are **expected** — same class as installing any self-built open-source app without a commercial certificate. This is not indicative of malware; verify artifacts via release tag, commit SHA, and optional checksums on the Release page.

---

## Known limitations (expected at ship)

- APK may remain **unsigned** if keystore secrets are not configured.
- Mobile UI remains WebView shell; **no device smoke gate** for v1.5.4.
- iOS lane optional when signing secrets absent.
