## Summary (planned)

First **mobile production** milestone on the **same `1.5.5` version** as desktop: signed Android builds, device smoke gates, and incremental native shell work — without forking the shared messaging kernel.

---

## Planned highlights

### Mobile production

- Release-signed APK (GitHub secrets or maintainer keystore).
- Device matrix M1–M3 documented in `docs/assets/demo/v1.5.3/mobile-verification.md`.
- 16 KB native page-size alignment if required by target devices.

### Version parity

- `versionName` / in-app label match desktop for tag `v1.5.5`.
- No separate mobile semver line.

### Native (incremental)

- Push / background hooks as needed for smoke (best-effort M4).
- Kotlin/Swift adapters remain thin — logic stays in shared TS.

---

## Non-goals

- Full mobile UI rewrite.
- iOS App Store unless signing scope opens explicitly.
