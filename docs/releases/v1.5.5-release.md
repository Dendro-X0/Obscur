---

## Summary

Desktop/web **refinement** patch on the shared kernel: honest relay publish feedback, account projection recovery after relay heal, faster vault cold open on large message DBs, and visible degraded relay state in Settings → Relays.

**Mobile production** (signed APK, store gates) remains **postponed** — same version line, optional preview APK on tag when CI runs.

---

## Added

- **`relay-publish-user-copy`** — maps `reasonCode` (`no_writable_relays`, `quorum_not_met`, `relay_degraded`) to user-facing strings; DM send toasts on confirmation failure.
- **`RelayReadinessSettingsBanner`** — Settings → Relays shows transport degraded/offline copy, writable counts, and manual recovery retry.
- **Vault progressive scan** — `onCandidatesBatch` during cursor walk; first grid paint before full DB scan; capped local URL enrich (48 items, then idle).

---

## Changed

- **DM send (R1)** — no optimistic “sent” without wire/relay evidence; background confirmation drives `sending` → `accepted` / `failed`.
- **Account projection (R2)** — re-bootstraps degraded projection when `relayRecovery` returns to `healthy` with writable relays.
- **Settings (U2)** — amber nav indicator on Relays when transport is not healthy.

---

## Known limitations

- Release APKs may remain unsigned without Android keystore secrets.
- Mobile UI remains WebView shell; device matrix not a gate for this tag.
- iOS / F-Droid / store distribution optional per [indie distribution policy](../program/v1.5.x-feature-roadmap.md).
