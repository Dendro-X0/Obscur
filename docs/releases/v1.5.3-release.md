## Summary (planned)

v1.5.3 continues **desktop/web smoothness** and starts **mobile release readiness** in the same monorepo:

- Deeper Vault / heavy-page cold start improvements.
- Long-session stability fixes with regression tests.
- **Mobile shell** build variant for Tauri Android (`NEXT_PUBLIC_MOBILE_SHELL`).
- Documented path to dedicated mobile UI in v1.5.4+ without forking kernel logic.

---

## Planned highlights

### Client smooth (desktop + web)

- Vault aggregation performance on large local databases.
- Search/settings deferral where still cold.
- Targeted lifecycle/relay stability fixes.

### Mobile foundations

- `build-pwa-shell.mjs` — `desktop` vs `mobile` static exports.
- CI Android builds mobile-shell bundle.
- Device verification matrix (not browser-only).

---

## Known limitations (expected at ship)

- Mobile UI remains **responsive shell + tab bar** until v1.5.4 DM-first layouts.
- iOS may remain CI-optional depending on signing.
- Browser mobile simulation is **not** a release gate.
