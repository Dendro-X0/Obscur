## Why tag push does not create a GitHub Release (R13)

**Tag push** (`git push origin v1.8.14`) runs **build + verify only**. The **Publish Release** job is **skipped** unless you use **workflow_dispatch** with **`publish_release: true`**.

```yaml
# .github/workflows/release.yml — publish-release job
if: workflow_dispatch && inputs.publish_release == true
```

So a 40-minute green workflow with **Publish Release — Skipped** is **expected** on tag push. GitHub **Latest** stays **v1.8.11** until a manual dispatch publish (maintainer chose to defer; see handoff).

## Tag workflow (build-only, optional)

```bash
pnpm version:check && pnpm docs:check
pnpm release:test-pack -- --skip-preflight
git tag -a v1.8.14 -m "v1.8.14 — batch exit build cache"
git push origin v1.8.14
```

**Artifacts:** CI uploads from verify; no GitHub Release page unless publish step runs.

**Publish (only if you want Latest to move):** Actions → **Obscur Full Release** → **Run workflow** → check **`publish_release`** → set **`release_tag`** = `v1.8.14` → run after verify green. **Not required** for v1.9.x development.
