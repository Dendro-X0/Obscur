## Highlights

### Managed workspace (Test 8)

- **Genesis publish** — workspace community creation succeeds when relay whitelist is open (dev `nostr-rs-relay.toml`).
- **Invite → accept → relay join** — A/B flow completes; acceptance recorded in DM; sealed group chat active on both clients.
- **Group chat header** — member count, online count (presence evidence), last activity timestamp under title.
- **Message persistence** — hydrate from local store on open (`loadPersistedSealedGroupMessages`); Tauri SQLite when available, else scoped chat-state.
- **Leave / roster** — terminal membership cache; participants modal shows LEFT/EXCLUDED; header prefers live CRDT minus terminal lists.

### Relay / activation hardening (session carry-forward)

- Loopback WebSocket in Tauri; alias-aware relay pool; ephemeral publish fallback.
- Coordination CORS preflight fix; honest create/join copy (no false “relay optional” when localhost relay configured).
- Member count merge fixes in `group-provider` (no stale `Math.max` on leave).

## Install

Desktop installer attached to the GitHub Release for tag `v1.8.8` (Windows/macOS/Linux per workflow matrix). Mobile artifacts optional per [mobile-desktop-version-policy.md](../program/mobile-desktop-version-policy.md).

**Local managed-workspace dev:**

```bash
pnpm -C apps/coordination dev
pnpm dev:relay:docker   # or pnpm dev:relay when Docker available
pnpm dev:desktop:online
```

Copy `apps/pwa/.env.example` to `.env.local` and set `NEXT_PUBLIC_COORDINATION_URL=http://127.0.0.1:8787`, workspace relay `ws://localhost:7000`.
