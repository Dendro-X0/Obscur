# v0.8.6 Ticketized Roadmap (Setup + Configuration UX)

_Last reviewed: 2026-03-06 (baseline commit a6f71d2)._

_Drafted: 2026-03-06._

## 1) Release Theme

1. Mainstream-friendly setup by default.
2. Power-user depth without clutter.
3. Reliable, explicit save/publish behavior under unstable relay conditions.

## 2) Product Principles

1. **Simple First**: one primary action per settings surface.
2. **Progressive Disclosure**: advanced controls are available but collapsed by default.
3. **Visible State**: every save/publish action shows `idle | in-progress | success | failed`.
4. **Recoverability**: failed actions provide direct retry guidance.

## 3) Scope

### In scope

1. General category redesign pass:
   - Profile
   - Appearance
   - Notifications
   - Identity
   - Security
2. Shared settings save-state UX primitives.
3. Relay-aware reliability improvements for settings publish flows.
4. Docs/runbook updates for maintainers.

### Out of scope

1. Protocol/network contract changes.
2. New social/messaging feature surfaces.
3. DB schema migrations.

## 4) Tickets

### A. Framework (P0)

1. `V086-SETUP-001` Settings Save-State Contract
   - Add shared internal model for settings action state:
     - `idle | waiting | preparing | publishing | success | failed`
   - Surface standard UI hints and retry patterns.
   - Acceptance:
     - all touched settings actions use the same status pattern.

2. `V086-SETUP-002` Basic/Advanced Surface Structure
   - Keep mainstream defaults in main panel.
   - Move deep controls under explicit `Advanced` toggles where applicable.
   - Acceptance:
     - no critical action requires entering advanced mode.

### B. General/Profile (P0)

1. `V086-PROFILE-001` Publish Pipeline Status UX
   - Show profile publish phases:
     - waiting for relays,
     - mining/signing,
     - publishing,
     - success/failure.
   - Include relay publish summary (`successCount / totalRelays`) when available.
   - Acceptance:
     - users can see why publish is waiting/failing without opening console.

2. `V086-PROFILE-002` Relay-Flap Resilience
   - Retry profile publish with bounded backoff on transient no-relay errors.
   - Prefer publish to enabled relay set, then fallback path.
   - Acceptance:
     - short relay disconnect windows no longer fail immediately.

3. `V086-PROFILE-003` Profile Validation + Clarity
   - Improve inline validation for username/NIP-05/avatar URL.
   - Keep clear helper text and deterministic error messages.
   - Acceptance:
     - invalid fields are caught before publish attempts.

### C. General/Appearance (P1)

1. `V086-APPEAR-001` Appearance Quick Controls
   - Ensure language/theme controls provide immediate visual feedback.
   - Add reset-to-default per section.
   - Acceptance:
     - changes are reflected instantly and are reversible.

2. `V086-APPEAR-002` Accessibility Subpanel
   - Add text scale / reduced motion / contrast assist options.
   - Acceptance:
     - options persist and are reflected in UI behavior.

### D. General/Notifications (P1)

1. `V086-NOTIF-001` Notification Granularity
   - Replace single action with toggles:
     - DM messages
     - mentions/replies
     - invites/system alerts
   - Acceptance:
     - user can selectively enable channels.

2. `V086-NOTIF-002` Permission Diagnostics + Test
   - Show permission state (`granted | denied | default | unsupported`).
   - Add test notification action.
   - Acceptance:
     - user can verify delivery without external triggers.

### E. Docs/Ops (P1)

1. `V086-DOC-001` Setup/Config Maintainer Notes
   - Update docs with behavior contracts, failure modes, and troubleshooting.
2. `V086-OPS-001` Release Notes + QA Matrix
   - Add v0.8.6 targeted QA matrix for settings reliability.
   - Closure reference: [v0.8.6 Settings Maintainer Notes + QA Matrix](./29-v0.8.6-settings-maintainer-notes-and-qa-matrix.md).

### F. Account/Identity (P0)

1. `V086-IDENT-001` Identity Overview + Storage Mode
   - Show `npub`, hex pubkey, copy actions, and key storage mode badge (`native | encrypted_local | session_only | unknown`).
2. `V086-IDENT-002` Re-authenticated Key Reveal
   - Require challenge before reveal and auto-hide private key after 20 seconds.
   - Hide on blur and keep explicit copy/export actions.
3. `V086-IDENT-003` Identity Integrity Diagnostics
   - Display stored/derived/native session keys and classify `ok | mismatch | unknown`.
   - Block key reveal when mismatch is detected and surface recovery guidance.
4. `V086-IDENT-004` Hardened Danger Zone
   - Split clear-local-data and wipe-account paths.
   - Add typed confirmation + countdown gate before wipe-account confirmation.

### G. Account/Security (P0)

1. `V086-SEC-001` Security Posture Summary
   - Add `strong | moderate | weak` posture strip derived from active controls.
2. `V086-SEC-002` Unified Security Control Status
   - Reuse settings action-status contract for security control updates.
3. `V086-SEC-003` Auto-Lock Effective Behavior
   - Show explicit behavior copy (“Locks after X inactivity”).
4. `V086-SEC-004` Capability Diagnostics
   - Surface `supported | unavailable | error` style capability labels for clipboard/biometric/Tor.
5. `V086-SEC-005` Session Controls
   - Add `Lock now` and retain clear local data in session management.

## 5) Test Plan

### Unit

1. Publish phase state transitions.
2. Retry classification for transient relay failures.
3. Validation helpers for profile fields.

### Integration

1. Profile publish during relay flap/reconnect.
2. Notification permission transitions + test action behavior.
3. Appearance toggles persist across app restart.
4. Identity reveal requires challenge and expires after 20 seconds.
5. Security posture updates as controls change.
6. Session `Lock now` transitions state immediately.

### Manual

1. First-run settings walkthrough on desktop + mobile layouts.
2. Light/dark contrast checks on touched controls.
3. Offline/online publish attempts produce expected status and recovery hints.

## 6) Sequencing

1. Wave 1 (P0): `SETUP-001`, `PROFILE-001`, `PROFILE-002`.
2. Wave 2 (P1): `PROFILE-003`, `NOTIF-001`, `NOTIF-002`.
3. Wave 3 (P0): `IDENT-001` ... `IDENT-004`, `SEC-001` ... `SEC-005`.
4. Wave 4 (P1): docs/ops updates.

## 7) Exit Criteria

1. General/Profile shows explicit publish state and relay summary.
2. Relay transient failures are retried with bounded policy.
3. No red-overlay class errors for expected transient settings failures.
4. Settings actions provide deterministic user-facing recovery guidance.
5. Account/Identity and Account/Security meet light/dark parity and mobile/desktop usability baseline.
