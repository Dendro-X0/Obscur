# v0.9 Rescue Wave 0: Relay/NIP API and Function Audit Matrix

_Last reviewed: 2026-03-08 (baseline commit ab08104)._

## Purpose

This is the canonical Wave 0 inventory for relay/NIP pathways used by core P2P flows across desktop and web-dev.  
It tracks what is currently `working | flaky | broken | dead`, the failure mode, and the owner path for recovery.

## Classification Rubric

1. `working`: passes expected behavior with deterministic outcomes under normal conditions.
2. `flaky`: works in some runs but fails under realistic relay/network/provider churn.
3. `broken`: code path executes but cannot complete core behavior reliably.
4. `dead`: path is no longer reachable or intentionally disabled.
