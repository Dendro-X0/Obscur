# 27 v1.0.8 CP3 Voice Replay Matrix

_Last reviewed: 2026-03-27 (baseline commit 458c74c7)._

This matrix defines the manual weak-network replay evidence required before `v1.0.8` release preflight/tagging (`CP3` in `docs/25-versioned-phase-plan-v1.0.7-v1.0.9.md`).

## Scope

1. Small-room realtime voice transition continuity (`create/join -> connecting -> active/degraded/ended`).
2. Degraded-path determinism under weak network conditions.
3. Unsupported-path diagnostics clarity on unsupported runtime/capability paths.
4. One-copy incident bundle quality for reproducible triage.

## Session Setup

1. Use one desktop runtime and one web runtime.
2. Use two test accounts (`A`, `B`) and one target room/community.
3. Keep console open on the active runtime.
4. Ensure latest code with `window.obscurM6VoiceCapture` installed.
5. Confirm replay bridge is installed:
: `window.obscurM6VoiceReplay?.getState?.()`

## Replay Steps

1. Clear app-event buffer:
: `window.obscurAppEvents?.clear()`
2. Baseline probe:
: `window.obscurAppEvents?.getCrossDeviceSyncDigest?.(200)?.summary?.realtimeVoiceSession`
3. Run normal connect path replay:
: `A` starts or joins voice room,
: `B` joins,
: observe session becomes interactive (`connecting -> active`).
4. If realtime voice UI flow is not exposed in the current build, run deterministic bridge replay:
: `window.obscurM6VoiceReplay?.runWeakNetworkReplay?.()`
5. Run degraded-path replay:
: temporarily degrade network/relay quality,
: observe transition to degraded state,
: restore network and observe recovery behavior.
6. Run unsupported-path replay (where applicable):
: in an unsupported browser/runtime capability context, attempt voice session,
: verify explicit unsupported transition is emitted (not silent success).
7. Export one-copy replay + capture bundle (preferred):
: `copy(window.obscurM6VoiceReplay?.runWeakNetworkReplayCaptureJson?.({ clearAppEvents: true, captureWindowSize: 400 }))`
8. Capture-only export (fallback):
: `copy(window.obscurM6VoiceCapture?.captureJson(400))`
9. Optional account-switch replay export (for CP2+ continuity probes):
: `copy(window.obscurM6VoiceReplay?.runAccountSwitchReplayCaptureJson?.({ clearAppEvents: true, captureWindowSize: 400 }))`
10. Optional raw digest fallback export:
: `copy(JSON.stringify((() => {`
: `  const digest = window.obscurAppEvents?.getCrossDeviceSyncDigest?.(400);`
: `  return {`
: `    realtimeVoiceSession: digest?.summary?.realtimeVoiceSession ?? null,`
: `    asyncVoiceNote: digest?.summary?.asyncVoiceNote ?? null,`
: `    deleteConvergence: digest?.summary?.deleteConvergence ?? null,`
: `    transitions: digest?.events?.["messaging.realtime_voice.session_transition"] ?? [],`
: `    voiceNoteStartFailed: digest?.events?.["messaging.voice_note.recording_start_failed"] ?? [],`
: `    deleteRemoteResult: digest?.events?.["messaging.delete_for_everyone_remote_result"] ?? [],`
: `    recentWarnOrError: digest?.recentWarnOrError ?? [],`
: `    m0Triage: window.obscurM0Triage?.capture?.(300) ?? null,`
: `  };`
: `})(), null, 2))`
11. Connecting-timeout watchdog gate probe (new M6 timeout convergence helper):
: `copy(window.obscurM6VoiceReplay?.runConnectingWatchdogCaptureJson?.({ captureWindowSize: 400, expectedNoOpenRelay: true }))`
12. Gate-only probe (if you only need pass/fail checks and failed-check samples):
: `copy(window.obscurM6VoiceReplay?.runConnectingWatchdogGateProbeJson?.({ captureWindowSize: 400, expectedNoOpenRelay: true }))`

## Pass Criteria

1. Voice session transitions are explicit and deterministic (no silent optimistic success).
2. `realtimeVoiceSession.transitionCount > 0` for replay window.
3. `realtimeVoiceSession.riskLevel` is not `high` during recoverable weak-network replay.
4. Unsupported runtimes emit explicit unsupported reason codes.
5. No replay step produces unrecoverable blank-page/frozen-navigation state.
6. Watchdog gate pass/fail state aligns with timeout evidence (`openRelayCount`, RTC state, timeout diagnostics count).

## Escalation Criteria

Escalate as active blocker if either occurs:

1. `realtimeVoiceSession.recoveryExhaustedCount > 0` in expected recoverable network replay.
2. repeated unsupported transitions on previously supported runtime without capability changes.
3. transition stream missing entirely despite attempted voice lifecycle actions.

When escalating, attach:

1. `copy(window.obscurM6VoiceCapture?.captureJson(400))`
2. `copy(window.obscurM0Triage?.captureJson(300))`
3. `copy(window.obscurM6VoiceReplay?.runWeakNetworkReplayCaptureJson?.({ clearAppEvents: true, captureWindowSize: 400 }))`
4. `copy(window.obscurM6VoiceReplay?.runAccountSwitchReplayCaptureJson?.({ clearAppEvents: true, captureWindowSize: 400 }))`
5. `copy(window.obscurM6VoiceReplay?.runConnectingWatchdogCaptureJson?.({ captureWindowSize: 400, expectedNoOpenRelay: true }))`
6. short timestamped replay timeline (action, phase seen, failure point).
