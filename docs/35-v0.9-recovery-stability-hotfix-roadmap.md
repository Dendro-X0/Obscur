# v0.9 Recovery Roadmap: Stability Hotfix First

_Last reviewed: 2026-03-07 (baseline commit ab08104)._

## Objective

Restore reliable P2P onboarding immediately, then reintroduce deterministic discovery in controlled gates.

## Locked Decisions

1. Ship Wave A stability hotfix first.
2. Keep new discovery primitives in code, but hide unstable UI by default.
3. Friend-add must never depend on global search.
4. Web dev harness remains a first-class two-user test environment.

## Wave A: Stability Hotfix (Release Blocking)

1. Add Friend safe path:
`contact-card | npub | hex-pubkey -> resolve -> send request`.
2. Keep deterministic resolver/outbox code gated by flags:
- `stabilityModeV090=true` by default
- `deterministicDiscoveryV090=false` by default
3. Profile actions split:
- `Save Local` (local persistence only)
- `Publish Network` (relay publish with timeout guardrails)
4. Disable misleading API probe endpoint by default; relay panel remains advisory.
5. Remove infinite loader risk from request/publish paths using bounded timeout states.

## Wave B: Deterministic Discovery Reintroduction (Gated)

1. Step 1: Resolver-only UI.
2. Step 2: Resolver + outbox send.
3. Step 3: Full lifecycle UX.
4. Keep fixed resolver order:
`contact_card -> friend_code_v2 -> npub/hex -> legacy_code`.
5. Keep global people/community search as best-effort discovery only.

## Feature Flags and Rollback

1. `stabilityModeV090`
- Default `true`.
- Forces safe Add Friend UX.
2. `deterministicDiscoveryV090`
- Default `false`.
- Enable only after Wave B gate passes.

Rollback policy:
- Keep `stabilityModeV090=true` for hotfix branches.
- Do not enable deterministic UI in release branches without Wave B signoff.

## Release Gate Addendum (P2P Core Pack)

Before creating a release tag, pass the P2P core pack:

1. `apps/pwa` typecheck
2. Resolver/Friend Code tests
3. Request outbox tests
4. Profile publish timeout tests
5. Existing reliability/sync/storage focused suite

## Manual Acceptance Target

10/10 successful runs:

1. Desktop A + Web-dev B: resolve via contact-card/npub/pubkey.
2. Send request and accept.
3. Confirm contact usable for DM without hanging states.
