# v0.9.0 Ticketized Roadmap (Community + 1:1 UX)

_Last reviewed: 2026-03-05 (baseline commit ad788a4)._

v0.9.0 focuses on making Obscur feel like a daily-use tool for niche groups and work communication: clear information structure, fast retrieval, and coordination primitives, while preserving strong privacy and decentralization.

## 1) Release Theme

1. Community as a first-class product surface (not "just a group chat").
2. Local-first, private-by-default UX with explicit controls over metadata leakage.
3. Performance and operability at scale: smooth timelines, fast media, and fast retrieval.

## 2) Scope and Guardrails

### In scope

1. Community structure: spaces/channels, threads, and curated knowledge surfaces.
2. Member experience: roster, community-scoped identity, and governance policies.
3. Coordination primitives: lightweight polls/events/tasks as typed message payloads.
4. Local-first power features: search, bookmarks, media albums, file cabinet.
5. UX polish and performance budgets for chat/media/community flows.

### Out of scope

1. Protocol redesign, relay contract changes, or DB schema migrations that require network-wide coordination.
2. Broad whole-app visual redesign unrelated to messaging/community surfaces.
3. Complex "workspace suite" features (full kanban, full calendar system) unless explicitly scheduled.

## 3) Product Principles (Non-Negotiables)

1. Information architecture first: users always know where things belong.
2. Private by default: clear UI when something increases discoverability/metadata exposure.
3. Operable communities: moderation and coordination workflows must work for small teams.
4. Retrieval wins: search/pins/wiki/bookmarks are core UX, not extras.
5. No perf regressions: new surfaces must meet existing smooth-scroll budgets.

## 4) Ticket Backlog

### A. Foundations (P0)

1. `V090-DESK-001` Desktop Auth Layering Reliability
   - Resolve `DESK-AUTH-OVERLAY-001`: fullscreen auth overlay creates blocked/non-interactive background area in desktop runtime.
   - Define one authoritative auth-shell model:
     - either true fullscreen auth route, or separate fixed-size auth window behavior, but never hybrid overlay layers.
   - Add regression check for desktop auth startup to prevent recurrence.

2. `V090-COMM-001` Spaces/Channels
   - Add community "spaces" with scoped timelines (e.g., announcements, help, general).
   - Space-level unread and mention counts.
   - Space-level notification policy (client-side).

3. `V090-COMM-002` Threads (Minimum Viable)
   - "Reply in thread" + "view thread" UX.
   - Thread unread counters and jump-to-parent.
   - Keep thread rendering lightweight and virtualized.

4. `V090-KNOW-001` Pins/Pinboard
   - Pin messages/links/files per space.
   - Pin index UI: title + source + jump.
   - Optional "pinned rules" slot surfaced during onboarding.

5. `V090-UX-001` Community Onboarding (Template + Checklist)
   - Join link/QR preview using only public metadata.
   - Onboarding checklist: rules, intro, notifications, relays.
   - Community templates that create default spaces and baseline policies.

### B. Member Experience + Governance (P0/P1)

1. `V090-MEM-001` Member Roster (Privacy-Aware)
   - Member list with roles and join status.
   - Optional "last active" representation (off by default).
   - Fast-path rendering and incremental hydration.

2. `V090-ID-001` Community-Scoped Profile
   - Display name/avatar override per community.
   - Explicit control and explanation: global identity vs community alias.

3. `V090-GOV-001` Governance Policies
   - Policy modes: trusted operators vs consensus moderation vs hybrid.
   - UI that explains consequences (kick/leave/key rotation).
   - Keep enforcement client-side where possible (no protocol redesign).

### C. Coordination Primitives (P1)

1. `V090-COORD-001` Polls
   - Encrypted poll payloads with vote UX.
   - Close time and results view.

2. `V090-COORD-002` Events/Calendar Posts
   - Time + RSVP + reminders (client-side).
   - Link event to a thread for discussion.

3. `V090-COORD-003` Lightweight Tasks
   - Assign, status, due date, and link-to-thread.
   - Keep it minimal to avoid "project management suite" scope creep.

### D. Retrieval and Local-First Power Features (P0/P1)

1. `V090-SEARCH-001` Community Search (Local)
   - Search by space, sender, file type, and time window.
   - Fast UI with incremental results and no main-thread stalls.

2. `V090-SAVE-001` Bookmarks + Read Later
   - Per-user bookmarks (local-first).
   - Bookmark index with jump-to-message.

3. `V090-MEDIA-001` Media Albums Per Space
   - Gallery view for images/videos.
   - Quick-open in lightbox and jump-to-message.

4. `V090-FILES-001` File Cabinet
   - List shared files/docs with sender file name, type, size, and open/download actions.
   - Consistent UI across light/dark themes.

### E. 1:1 Interaction Improvements (P0/P1)

1. `V090-DM-001` Conversation Organization
   - Notes/labels for conversations (local-only).
   - Better "search within conversation".

2. `V090-DM-002` Quiet Coordination
   - Lightweight "poll/event/task" cards usable in DMs as well.

### F. Performance + Quality Gates (P0)

1. `V090-PERF-001` Performance Budget Enforcement
   - Define measurable budgets for:
     - timeline scroll FPS median,
     - p95 UI update latency,
     - media-heavy message rendering.
   - Add profiling scenarios to the maintainer runbook.

2. `V090-QA-001` Targeted Integration Coverage
   - Space switching, thread view, pins, and search.
   - Light/dark snapshots for new surfaces.

### G. Live Calling (P0/P1, Privacy-First Rollout)

1. `V090-CALL-001` Calling Architecture and Contracts (P0)
   - Add typed call signaling contract over existing relay messaging path:
     - invite, accept, reject, hangup, ICE candidate, call state updates.
   - Keep signaling encrypted and scoped to participants.
   - Add strict feature flag and kill switch for all call surfaces.

2. `V090-CALL-002` 1:1 Audio Call MVP (P0)
   - Implement 1:1 WebRTC audio calling for PWA + desktop.
   - Add reliable state machine UX:
     - ringing, connecting, connected, reconnecting, ended, failed.
   - Add media device controls:
     - mute/unmute mic, speaker output toggle, input selection.
   - Add timeout + retry behavior for unanswered/failed calls.

3. `V090-CALL-003` NAT Traversal and Fallback (P0)
   - Add STUN configuration defaults and TURN fallback support.
   - Keep policy explicit in settings:
     - direct-only mode, or relay-assisted mode.
   - Surface clear connection diagnostics without leaking sensitive metadata.

4. `V090-CALL-004` Call Privacy and Safety Guards (P0)
   - Add privacy controls:
     - allow calls from everyone / contacts-only / community members-only.
   - Add per-conversation and per-community call block/mute controls.
   - Ensure no call invite toast/overlay appears for blocked senders.

5. `V090-CALL-005` Video and Small Group Call Phase (P1, post-MVP)
   - Add optional 1:1 video after audio stability is proven.
   - Add small-group calls with explicit cap and quality targets.
   - Defer large group calling and SFU deployment to follow-up roadmap (v0.9.1+).

6. `V090-CALL-006` Call Quality Telemetry (P1)
   - Add local diagnostics for jitter/packet loss/RTT.
   - Track call setup success rate and drop reasons for release gating.
   - Keep metrics local or privacy-preserving by default.

## 5) Sequencing (Recommended)

1. P0: `COMM-001`, `COMM-002`, `KNOW-001`, `UX-001`
2. P0/P1: `MEM-001`, `ID-001`, `GOV-001`
3. P0/P1: `SEARCH-001`, `SAVE-001`, `MEDIA-001`, `FILES-001`
4. P0: calling foundation + 1:1 audio (`CALL-001`, `CALL-002`, `CALL-003`, `CALL-004`)
5. P1: coordination primitives (`COORD-*`) + DM polish (`DM-*`) + extended calling (`CALL-005`, `CALL-006`)
6. P0: perf/QA gates (`PERF-001`, `QA-001`) throughout, not at the end

## 6) Exit Criteria

1. Communities have structure (spaces + threads) and retrieval (pins + search).
2. Member experience is usable without sacrificing privacy (roster + scoped identity).
3. At least one coordination primitive shipped (polls recommended first).
4. 1:1 encrypted audio calling is stable behind feature flag with NAT fallback.
5. No performance regressions in heavy community timelines.
6. New surfaces maintain strict light/dark parity.
