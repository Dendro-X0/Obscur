# 28 v1.0.9 CP3 Anti-Abuse Replay Matrix

_Last reviewed: 2026-03-23 (baseline commit 563ef5d)._

This matrix defines deterministic replay evidence capture for M7 anti-abuse cooldown behavior.

## Goal

Capture reproducible evidence for:

1. `incoming_connection_request_peer_rate_limited`
2. `incoming_connection_request_peer_cooldown_active`
3. digest + capture readiness (`readyForCp3Evidence: true`)

## Preconditions

1. Dev server running with latest `main`.
2. `window.obscurM7AntiAbuseCapture` and `window.obscurM7AntiAbuseReplay` are installed.

Check:

```js
window.obscurM7AntiAbuseCapture?.capture?.(50)
window.obscurM7AntiAbuseReplay?.getLastReplay?.()
```

## Replay Steps

1. Run deterministic replay:

```js
window.obscurM7AntiAbuseReplay?.runPeerCooldownReplay({ clearAppEvents: true })
```

2. Export one-copy CP3 evidence bundle:

```js
copy(window.obscurM7AntiAbuseReplay?.runPeerCooldownReplayCaptureJson({ clearAppEvents: true }))
```

## Expected Evidence

1. `replay.attempts` contains blocked entries in this order:
   1. `incoming_connection_request_peer_rate_limited`
   2. `incoming_connection_request_peer_cooldown_active`
2. `replay.replayReadiness.readyForCp3Evidence === true`
3. `capture.antiAbuse.replayReadiness.readyForCp3Evidence === true`
4. `capture.antiAbuse.summary.peerRateLimitedCount >= 1`
5. `capture.antiAbuse.summary.peerCooldownActiveCount >= 1`

## Notes

1. This replay bridge exists for deterministic CP3 diagnostics in unstable/manual-constrained sessions.
2. It does not replace real runtime anti-abuse owner logic; it executes through canonical anti-abuse contracts.
