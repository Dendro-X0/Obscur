# v1.4.0 Closeout and Documentation Consolidation Contract

Last Updated (UTC): 2026-04-21  
Status: Active  
Release Lane: `v1.4.0`

## Purpose

This file defines what happens **after** `v1.4.0` is actually complete.

The goal is to:

1. remove or archive redundant version-based planning docs,
2. update canonical docs so the repo reflects post-`v1.4.0` truth,
3. prevent future contributors from resuming from outdated version-lane artifacts.

## Closeout Rule

Do not archive or remove version-based documentation until all are true:

1. the `v1.4.0` roadmap checklist is complete,
2. runtime evidence exists for the community-system guarantees and supporting blocker lanes,
3. `CHANGELOG.md` reflects the final shipped truth,
4. `docs/roadmap/current-roadmap.md` reflects the new steady state,
5. `docs/handoffs/current-session.md` contains a final closeout checkpoint.

## Canonical Docs To Update On v1.4.0 Completion

These files must be updated in the same closeout pass:

1. `CHANGELOG.md`
2. `docs/README.md`
3. `docs/roadmap/current-roadmap.md`
4. `docs/handoffs/current-session.md`
5. `docs/11-program-milestones-and-stability-history.md`
6. `docs/history/version-context.md`

## Version-Based Docs To Review for Archive/Consolidation

At `v1.4.0` closeout, explicitly review:

1. `docs/roadmap/v1.3.8-hybrid-offline-streaming-update-plan.md`
2. `docs/roadmap/v1.3.8-offline-ui-asset-inventory.md`
3. `docs/roadmap/v1.3.8-streaming-update-contract.md`
4. any temporary `v1.4.0` version-lane planning docs created during execution.

## Archive Rule

Move a version-based doc out of the active path only if:

1. its outcomes are reflected in canonical docs,
2. it is no longer needed as an execution artifact,
3. a replacement canonical reference is named in the archive commit.

## Required Closeout Checklist

- [ ] `v1.4.0` roadmap is marked complete with evidence.
- [ ] community-system mode guarantees are validated and reflected in canonical docs.
- [ ] `CHANGELOG.md` records the shipped truth.
- [ ] `docs/roadmap/current-roadmap.md` reflects post-`v1.4.0` priorities.
- [ ] `docs/README.md` points only to still-active version-lane docs.
- [ ] redundant version-based docs are archived or explicitly retained with reason.
- [ ] `docs/handoffs/current-session.md` contains the final closeout checkpoint.

## Post-Closeout Goal

After this process:

1. the repo should have fewer active version-lane docs,
2. the canonical docs should carry the durable truth,
3. future contributors should start from canonical files first, not stale lane plans.
