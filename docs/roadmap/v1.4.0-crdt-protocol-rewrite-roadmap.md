## Executive Summary

This roadmap supersedes the previous v1.4.0 in-place rewrite plan. After 2+ weeks stalled at v1.3.16 with unresolved severe blockers, the evidence suggests the current centralized-pattern architecture cannot be incrementally fixed. This document proposes a **fundamental architectural shift** to CRDT-native, gossip-based, content-addressed protocols.

**The Core Thesis:**
> Obscur is a P2P encrypted communication platform built on relay infrastructure. We've been applying centralized patterns (single truth, drift detection, projections) to a distributed system. This roadmap corrects that fundamental mismatch.

## Why Replace the Previous v1.4.0 Plan

### Evidence from Blocker Registry (as of 2026-04-24)

| Blocker | Root Cause | Pattern Fix |
|---------|------------|-------------|
| BLK-001: Media clears after login | Snapshot replacement overwrites state | **CRDT merge** preserves all evidence |
| BLK-002: B→A DM visibility gap | "Which source wins?" is wrong question | **CRDT union** merges all sources |
| BLK-005: Self-only community roster | Snapshot thinning on restore | **OR-Set** preserves concurrent adds |
| BLK-007: Live join/leave drift | Polling for state changes | **Gossip** provides epidemic broadcast |
| Ghost calls | Event replay as commands | **LWW-Register** with TTL for call state |

**Conclusion:** The blockers are not implementation bugs—they are architectural mismatches. Incremental fixes will continue to fail because the underlying paradigm is wrong.

## New Architectural Pillars

### Pillar 1: CRDT-Native State Containers

Replace all "snapshot → projection → UI" flows with CRDT-derived state:

```
Current (Broken):
Events → Reducer → Projection → UI (replaces state)

Proposed (Fixed):
Events → CRDT Container → Derived View → UI (merges state)
```

| Domain | Current Pattern | CRDT Type | Owner File |
|--------|----------------|-----------|------------|
| Member lists | Array replacement | **OR-Set** | `community-membership-crdt.ts` |
| Message history | Hydration filter | **LWW-Element-Set** | `chat-history-crdt.ts` |
| Presence | Binary online/offline | **G-Counter + decay** | `presence-crdt.ts` |
| Call state | Event replay | **LWW-Register + TTL** | `call-state-crdt.ts` |
| Profile metadata | Local override | **LWW-Register** | `profile-metadata-crdt.ts` |

### Pillar 2: Gossip Protocol for Real-Time

Replace polling with epidemic broadcast:

```typescript
// Current (Polling - expensive, laggy)
subscribeToEvents({ kinds: [ALL], authors: [...everyone] })

// Proposed (Gossip - efficient, eventually consistent)
interface GossipMessage {
  type: 'presence' | 'member_update' | 'history_sync';
  payload: SignedGossipEnvelope;
  ttl: number;              // Prevents floods
  vectorClock: VectorClock; // Causality tracking
}
```

**Hybrid Architecture:**
- **Gossip layer:** Real-time presence, member updates, lightweight signals
- **Nostr relays:** Message durability, backup, offline catch-up
- **Relays as bootstrap:** Gossip mesh formation via relay-mediated signaling

### Pillar 3: Content-Addressable Storage (CAS)

Separate media sync from message sync:

```typescript
// Current (Breaks when message sync uncertain)
media: { messageId: string, url: string }

// Proposed (Resilient, deduplicated)
media: {
  sha256: string;           // Content address
  size: number;
  mimeType: string;
  sources: Set<Pubkey>;     // Who has this hash
  fetchStatus: 'pending' | 'fetching' | 'complete';
}

// Sync: Ask peers "Do you have hash X?"
// UI: "Fetching media 3/5..." progress indicator
```

**Add Merkle Trees for Verification:**
```typescript
interface ConversationHead {
  conversationId: string;
  merkleRoot: string;      // Root of message hashes
  messageCount: number;    // Quick divergence detection
  vectorClock: VectorClock;  // Causality, not just time
}
```

```typescript
// New canonical owner
export const communityMembershipCrdt = {
  // Event → CRDT operation
  applyEvent: (set: ORSet<Pubkey>, event: CommunityEvent): ORSet<Pubkey> => {
    switch (event.kind) {
      case 'member_joined':
        return addToORSet(set, event.pubkey);
      case 'member_left':
      case 'member_removed':
        return removeFromORSet(set, event.pubkey);
      default:
        return set;
    }
  },
  
  // Merge concurrent states (the fix for "thinning")
  merge: (local: ORSet<Pubkey>, remote: ORSet<Pubkey>): ORSet<Pubkey> =>
    mergeORSets(local, remote),
  
  // Query current members
  getMembers: (set: ORSet<Pubkey>): Set<Pubkey> =>
    queryORSet(set)
};
```

### Phase 4: Call State CRDT (Weeks 9-10)

**Goal:** Fix ghost calls with LWW-Register + TTL

#### 4.1 Call State Container

**New File:** `apps/pwa/app/features/messaging/services/call-state-crdt.ts`

```typescript
interface CallStateContainer {
  callId: string;
  participants: ORSet<Pubkey>;
  status: LWWRegister<'inviting' | 'connected' | 'ended'>;
  startedAt: LWWRegister<number>;
  endedAt: LWWRegister<number | null>;
  
  // Auto-end stale calls (prevents ghost calls)
  isActive(): boolean {
    const status = this.status.value;
    const endedAt = this.endedAt.value;
    const startedAt = this.startedAt.value;
    
    if (status === 'ended') return false;
    if (endedAt !== null) return false;
    
    // Auto-end after max duration (2 hours)
    const MAX_CALL_DURATION = 2 * 60 * 60 * 1000;
    if (Date.now() - startedAt > MAX_CALL_DURATION) return false;
    
    return true;
  }
}
```

**Ghost Call Prevention:**
- Historical call events don't reactivate calls
- Call state is CRDT-derived, not event-replayed
- TTL ensures calls end even if "end" event is lost

### Phase 5: DM History CRDT (Weeks 11-12)

**Goal:** Fix BLK-002, BLK-003 with LWW-Element-Set for messages

#### 5.1 Chat History Container

**New File:** `apps/pwa/app/features/messaging/services/chat-history-crdt.ts`

```typescript
interface ChatHistoryContainer {
  // LWW-Element-Set: All messages, last writer wins on conflict
  messages: Map<MessageId, {
    content: Message;
    timestamp: number;
    vectorClock: VectorClock;
    source: DeviceId;
  }>;
  
  // Merge with remote (preserves all, resolves conflicts)
  merge(other: ChatHistoryContainer): ChatHistoryContainer {
    const merged = new Map(this.messages);
    
    for (const [id, msg] of other.messages) {
      const existing = merged.get(id);
      if (!existing || vectorCompare(msg.vectorClock, existing.vectorClock) > 0) {
        merged.set(id, msg); // LWW by vector clock
      }
    }
    
    return { messages: merged };
  }
  
  // Derive ordered view
  getOrderedMessages(): Message[] {
    return [...this.messages.values()]
      .sort((a, b) => vectorCompare(a.vectorClock, b.vectorClock));
  }
}
```

**Why this fixes BLK-002 (B→A visibility gap):**
- Current: "Which source wins?" → wrong question
- New: "Union all sources" → B's messages merge with A's

**Why this fixes BLK-003 (outgoing-only bias):**
- Current: Projection fallback triggers on sparse evidence
- New: CRDT preserves all messages from all sources, no thinning

## Risk Assessment & Mitigation

| Risk | Likelihood | Impact | Mitigation |
|------|------------|--------|------------|
| CRDT performance degrades with scale | Medium | High | Delta-state encoding, periodic compaction, tombstone limits |
| Gossip floods network | Low | High | TTL limits, fanout constraints, relay rate limiting |
| Vector clock overflow | Low | Medium | Periodic clock resets with sync points |
| Migration breaks existing data | Medium | Critical | Backward-compatible readers, migration scripts, feature flags |
| Mobile battery drain from gossip | Medium | High | Adaptive gossip rate, background relay only, battery-aware backoff |

**Rollback Plan:**
1. Feature flag `useCRDT: boolean` for gradual rollout
2. Maintain old snapshot paths as compatibility bridges
3. Automated tests validate both paths produce equivalent results
4. If CRDT path fails, fallback to snapshot path within 24h
