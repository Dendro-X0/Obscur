# v0.8.3 Ticketized Roadmap

_Last reviewed: 2026-03-05 (baseline commit ba23d79)._

This roadmap targets UX polish for chat/media with release-pipeline reliability as a hard gate.

## 1) Release Theme

1. UX-first refinement for media and chat interaction surfaces.
2. Deterministic release operations to prevent blocked tags/artifacts.
3. Feature-flagged rollout with strict light/dark parity and no perf regressions.

## 2) Scope and Guardrails

### In scope

1. Media/lightbox/carousel interaction polish behind `chatUxV083`.
2. Shared tokenization cleanup for touched chat/media controls.
3. Release preflight tooling and artifact verification hardening.
4. Release dry-run support without accidental publish.

### Out of scope

1. Protocol changes, schema migrations, or relay API contract changes.
2. New product feature surfaces unrelated to chat/media UX.

## 3) Ticket Backlog

### A. Operations (P0)

1. `V083-OPS-001` Tag Collision Guardrails
   - Add `release:preflight` script:
     - remote tag non-existence check,
     - branch/tag context validation,
     - `version:check` + `docs:check`,
     - required workflow/path sanity.

2. `V083-OPS-002` Deterministic Artifact Matrix
   - Required artifacts:
     - Windows installer (`.exe` or `.msi`)
     - macOS installer (`.dmg`)
     - Linux installer (`.AppImage` or `.deb`)
     - Android APK (`.apk`)
     - Android AAB (`.aab`, signed or unsigned)
   - Improve matrix logs and diagnostics.

3. `V083-OPS-003` Release Dry-Run Path
   - Ensure `workflow_dispatch` runs full build + artifact verification without publish by default.
   - Keep publish path explicit.

### B. UX Foundation (P1)

1. `V083-UX-001` Messaging UI Tokenization
   - Introduce reusable media-viewer control tokens/classes.
   - Apply to lightbox and carousel controls.

2. `V083-UX-002` Unified Media Interaction Model
   - Shared interaction utility/types for:
     - index navigation,
     - zoom clamp/capability flags,
     - pinch and swipe normalization.

### C. UX Polish (P1)

1. `V083-UX-003` Lightbox Redesign
   - Cleaner control cluster (`X`, `+`, `-`, reset).
   - Wheel + pinch zoom, drag pan when zoomed.
   - Keyboard support: `Esc`, arrows, `+/-`.

2. `V083-UX-004` Carousel and Container Polish
   - Improved arrows, indexing, transitions.
   - Better contrast hierarchy in message media container.
   - Preserve separate audio/file layouts.

3. `V083-UX-005` Rollout Wiring
   - Add `chatUxV083` setting, default `false`.
   - Keep v0.8.2 UX path as fallback when flag is disabled.

## 4) Suggested Sequencing

1. Wave 1: `OPS-001/002/003`
2. Wave 2: `UX-001/002`
3. Wave 3: `UX-003/004/005`
4. Wave 4: stabilization, docs/changelog, release dry-run

## 5) Exit Criteria

1. No tag-collision surprise before release.
2. Required artifact matrix passes in CI.
3. `chatUxV083` path is complete and behind flag with fallback intact.
4. Light + dark parity confirmed for touched UI.
5. No performance regression in heavy media chat scenarios.
