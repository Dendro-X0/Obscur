# 31 v1.0.10 CP3 Community Replay Matrix

_Last reviewed: 2026-03-23 (baseline commit c446ca8)._

This matrix defines deterministic replay evidence capture for M8 community lifecycle convergence and membership sendability checks.

## Goal

Capture reproducible evidence for:

1. community lifecycle convergence digest signals,
2. membership/sendability digest signals,
3. joined-membership room-key mismatch visibility,
4. replay + capture readiness (`readyForCp2Evidence: true`).

## Preconditions

1. Dev server running with latest `main`.
2. `window.obscurM8CommunityCapture` and `window.obscurM8CommunityReplay` are installed.

Check:

```js
window.obscurM8CommunityCapture?.capture?.(50)
window.obscurM8CommunityReplay?.getLastReplay?.()
```

## Replay Steps

1. Run deterministic convergence replay:

```js
window.obscurM8CommunityReplay?.runConvergenceReplay({ clearAppEvents: true })
```

2. Export one-copy CP3 evidence bundle:

```js
copy(window.obscurM8CommunityReplay?.runConvergenceReplayCaptureJson({ clearAppEvents: true }))
```

## Expected Evidence

1. `replay.emittedEvents.membershipRecoveryHydrateCount >= 1`.
2. `replay.emittedEvents.membershipLedgerLoadCount >= 1`.
3. `replay.emittedEvents.roomKeyMissingSendBlockedCount >= 1`.
4. `replay.latestDigestSummary.communityLifecycleConvergence.recoveryRepairSignalCount >= 1`.
5. `replay.latestDigestSummary.membershipSendability.roomKeyMissingSendBlockedCount >= 1`.
6. `replay.replayReadiness.observedJoinedRoomKeyMismatch === true`.
7. `replay.replayReadiness.readyForCp2Evidence === true`.
8. `capture.community.replayReadiness.readyForCp2Evidence === true`.

## Notes

1. This replay bridge is a deterministic CP3 evidence helper for account-switch/recovery lanes where manual timing is noisy.
2. It does not replace canonical community owner logic; it emits through typed app-event contracts and validates via digest/capture surfaces.
