## Local reproduction (Windows)

### Observed failure (before cache repair)

`pnpm ci:local:desktop-win` / `pnpm desktop:package` failed compiling dependency crate **`cc` 1.2.65**:

```text
error[E0583]: file not found for module `apple`
  --> …/.cargo/registry/src/…/cc-1.2.65/src/target.rs
```

Also missing under `cc-1.2.65/src/target/`: `generated.rs`, `llvm.rs`, `parser.rs`.

**Mechanism:** incomplete/corrupt Cargo registry extract (local cache), not an Obscur source bug. Pull chain: `tauri-build` → `tauri-winres` → `embed-resource` → `cc`.

### Mitigation applied locally

1. Removed `cc-1.2.65` from registry `src/` + `cache/`
2. Re-fetched via `cargo check` in `apps/desktop/src-tauri`
3. Confirmed `src/target/{apple,generated,llvm,parser}.rs` present
4. **`pnpm ci:local:desktop-win` → PASS** in 742.9s (2026-07-18)

## Android failure class (iteration 3 — confirmed from maintainer log)

```text
error[E0432]: unresolved import `keyring`
 --> src/native_keychain.rs:8:5
use keyring::Entry;
```

**Mechanism:** `Cargo.toml` already scopes `keyring` to `cfg(not(target_os = "android"))`, and all keychain *functions* were Android-stubbed, but the top-level `use keyring::Entry` was **not** cfg-gated — Android `cargo build --lib` fails before stubs matter.

**Fix:** gate the import with `#[cfg(not(target_os = "android"))]` in `native_keychain.rs`.

**Proof:**
- Host: `cargo check -p obscur_desktop_lib` → PASS
- Android: full local check needs NDK clang (failed ToolNotFound locally); CI Android job is the L3 proof after push/retag
- Unused-variable warnings in the log are **out of scope** this iteration
